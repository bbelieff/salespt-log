# 상태 전이 다이어그램

## 1. Meeting 상태 전이

```mermaid
stateDiagram-v2
    [*] --> 예약 : 사용자가 미팅 예약 생성
    
    예약 --> 완료 : 미팅 진행 완료 (사용자 액션)
    예약 --> 취소 : 미팅 취소 (사용자 액션)
    
    완료 --> 계약성사 : 계약 체결 (사용자 입력)
    완료 --> 계약실패 : 계약 미성사 (사용자 입력)
    
    계약성사 --> [*] : 수임비 입력 완료
    계약실패 --> [*] : 미팅 종료
    취소 --> [*] : 취소 처리 완료
    
    note right of 예약 : 초기 상태\n앱_미팅예약 시트에 추가
    note right of 완료 : 일정·계약관리 탭에서\n상태 업데이트
    note right of 계약성사 : 계약여부 = true\n수임비 금액 입력
```

## 2. Payment 상태 전이

```mermaid
stateDiagram-v2
    [*] --> 미승인 : 승인 대기 중인 건 발생
    
    미승인 --> 승인 : 기관 승인 완료 (시스템/사용자)
    미승인 --> 반려 : 승인 거절 (시스템)
    
    승인 --> 수납완료 : 실제 입금 확인 (사용자 확인)
    승인 --> 수납대기 : 입금 지연 상태
    
    수납대기 --> 수납완료 : 지연된 입금 완료
    수납대기 --> 수납취소 : 입금 취소/환불
    
    수납완료 --> [*] : 최종 완료
    반려 --> [*] : 처리 종료
    수납취소 --> [*] : 취소 처리 완료
    
    note right of 미승인 : 수납관리 탭\n승인건수 입력
    note right of 승인 : 승인 처리 후\n승인건수 증가
    note right of 수납완료 : 수납건수, 수납금액\n최종 입력
```

## 3. DailyEntry 저장 상태 전이

```mermaid
stateDiagram-v2
    [*] --> draft : 사용자가 데이터 입력 시작
    
    draft --> 검증중 : 저장 버튼 클릭 (사용자 액션)
    
    검증중 --> 검증실패 : 필수값 누락/형식 오류 (시스템 검증)
    검증중 --> 저장중 : 검증 통과 (시스템)
    
    검증실패 --> draft : 사용자가 오류 수정
    
    저장중 --> saved : Google Sheets 저장 성공 (시스템)
    저장중 --> 저장실패 : 네트워크/권한 오류 (시스템)
    
    저장실패 --> draft : 사용자가 재시도
    saved --> [*] : 저장 완료
    
    note right of draft : 프론트엔드 로컬 상태\n실시간 입력 반영
    note right of 검증중 : 클라이언트 측 유효성 검사\n필수값, 숫자 형식 등
    note right of 저장중 : API 호출 진행\nGoogle Sheets 업데이트
```

## 4. DBOrder 상태 전이

```mermaid
stateDiagram-v2
    [*] --> 입력 : 사용자가 DB 주문 입력
    
    입력 --> 확정 : 주문 확인 (사용자 액션)
    입력 --> 임시저장 : 임시 저장 (사용자 액션)
    
    임시저장 --> 입력 : 수정 재개 (사용자 액션)
    임시저장 --> 확정 : 주문 확정 (사용자 액션)
    임시저장 --> 취소 : 주문 취소 (사용자 액션)
    
    확정 --> 진행중 : 시스템에서 처리 시작
    확정 --> 취소 : 사용자 취소 요청
    
    진행중 --> 완료 : DB 공급 완료 (시스템/관리자)
    진행중 --> 취소 : 공급 중단 (시스템/관리자)
    
    완료 --> [*] : 주문 완료
    취소 --> [*] : 취소 처리 완료
    
    note right of 입력 : DB관리 탭에서\n수량/채널 입력
    note right of 확정 : 주문 확정 시\n재고에서 차감
    note right of 완료 : 실제 DB 전달 완료\n효율 지표에 반영
```

## 상태 전이 트리거 요약

### 사용자 액션 트리거
- **Meeting**: 예약 생성, 완료 처리, 계약 입력, 취소 요청
- **Payment**: 승인건수 입력, 수납 확인, 금액 입력
- **DailyEntry**: 데이터 입력, 저장 요청, 오류 수정
- **DBOrder**: 주문 입력, 확정 처리, 취소 요청

### 시스템 이벤트 트리거
- **Meeting**: Google Sheets 자동 집계 (TEXTJOIN)
- **Payment**: 기관 시스템 연동 (승인/반려)
- **DailyEntry**: 클라이언트 검증, API 응답
- **DBOrder**: 재고 관리 시스템, 관리자 처리

### 상태 전이 제약 조건
1. **Meeting**: 예약 → 완료 전환 시 date/time 검증 필수
2. **Payment**: 승인 → 수납완료 시 금액 일치성 검증
3. **DailyEntry**: 필수 필드 (date, channels) 검증
4. **DBOrder**: 재고 수량 확인 후 확정 처리
