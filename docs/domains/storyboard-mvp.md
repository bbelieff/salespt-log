# 스토리보드 — 세일즈PT 영업일지 MVP

> 두 사용자(수강생·트레이너)의 핵심 여정을 화면 단위로 기술.
> 각 화면은 `[M]` 모바일 / `[P]` PC / `[공통]` 으로 표시.

---

## 등장인물

| 페르소나 | 역할 | 주 디바이스 | 핵심 욕구 |
|---|---|---|---|
| **이수강** | PRM 5기 수강생 | 스마트폰 | "미팅 잡으면 바로 기록하고 싶다. 시트 열기 귀찮다." |
| **김트레이너** | 수석트레이너 | PC (가끔 모바일) | "수강생이 오늘 뭘 했는지 한눈에 보고 코칭하고 싶다." |

## 여정 1: 첫 접속 — 로그인 + 온보딩

```
[공통] S1-1  로그인
┌─────────────────────────┐
│                         │
│      세일즈PT 로고       │
│                         │
│  ┌───────────────────┐  │
│  │ 🔵 Google로 시작   │  │
│  └───────────────────┘  │
│                         │
│   "영업, 기록하면 늘어요"  │
│                         │
└─────────────────────────┘
```
- Google OAuth 팝업 → `calendar.events` scope 포함 동의
- 이미 온보딩 완료한 유저 → 바로 **S2-1 홈** 으로
- 처음인 유저 → **S1-2 온보딩** 으로

```
[공통] S1-2  온보딩 (첫 접속 1회만)
┌─────────────────────────┐
│  반갑습니다!              │
│  본인 정보를 선택해주세요  │
│                         │
│  기수  [ PRM 5기    ▼ ] │
│                         │
│  이름  [ 이수강      ▼ ] │
│  (선택한 기수의 수강생만   │
│   필터되어 표시)          │
│                         │
│  ┌───────────────────┐  │
│  │     시작하기       │  │
│  └───────────────────┘  │
└─────────────────────────┘
```
- 기수 선택 → 해당 기수 수강생 목록 필터 → 이름 선택
- "시작하기" → `registerUser(email, cohort, name, spreadsheetId)` 호출
- 마스터 레지스트리에서 매칭 → 본인 시트 귀속
- 완료 후 → **S2-1 홈**

## 여정 2: 일일 입력 (핵심 루프 — 매일 반복)

```
[M] S2-1  홈 (모바일)
┌─────────────────────────┐
│ 세일즈PT    이수강 ⚙️    │
├─────────────────────────┤
│                         │
│  LV.3 ⭐  1,250 XP     │
│  ████████░░░ → LV.4     │
│                         │
│  🔥 연속 12일째!         │
│                         │
├─────────────────────────┤
│  오늘의 기록 (4/16)      │
│                         │
│  생산   [ - ] 3 [ + ]   │
│  컨택   [ - ] 5 [ + ]   │
│  미팅   [ - ] 2 [ + ]   │
│  계약   [ - ] 0 [ + ]   │
│                         │
│  메모 [오전 A구역 방문_] │
│                         │
│  ┌───────────────────┐  │
│  │    💾 저장하기     │  │
│  └───────────────────┘  │
│                         │
├─────────────────────────┤
│  📊홈   📅미팅   👤MY   │
└─────────────────────────┘
```
- 숫자 입력: **+/- 스테퍼** (탭 한 번으로 1 증감) + **숫자 직접 입력** (숫자 누르면 키패드)
- 이미 오늘 기록이 있으면 해당 값 불러와 표시 (수정 모드)
- 저장 → 낙관적 UI (버튼 → ✅ 체크 애니메이션)
- XP 획득 토스트: `+45 XP! 🎉`
- 스트릭 카운트 +1 갱신

### 저장 시 데이터 흐름
```
[앱] 저장 클릭
  → service.logToday(user, { date, production, contact, meeting, contract, note })
    → repo.appendEntry(spreadsheetId, entry)     // 앱_일일입력 탭에 행 추가
    → 시트 수식이 성과관리 탭 자동 갱신           // 앱은 안 건드림
  ← { ok: true, xp: 45 }
[앱] XP바 애니메이션 + 토스트
```

- 상단: **월 스와이프 네비** (좌우 스와이프 또는 ◀▶ 버튼)
- 날짜 셀: 미팅 건수만큼 **점(dot)** 표시
- 날짜 탭 → 하단에 해당일 미팅 목록 표시
- 채널은 **작은 배지**로 표시 (시트 rendered에는 미포함)
- **"+ 미팅 추가"** 또는 빈 날짜 탭 → **S3-2 바텀시트** 올라옴

```
[M] S3-2  미팅 예약 바텀시트 (날짜 선택 후)
┌─────────────────────────┐

│                         │

│                         │
│  ⏰ 시간                 │
│  ┌──────────────────┐   │
│  │   13 : 00        │   │ ← 15분 단위 선택 또는 직접 입력
│  └──────────────────┘   │
│                         │
│  📂 채널                 │
│  ┌──────────────────┐   │

│  └──────────────────┘   │
│                         │
│  🏢 업체명 *             │
│  ┌──────────────────┐   │
│  │ ○○치킨           │   │
│  └──────────────────┘   │
│                         │
│  📍 지역                 │
│  ┌──────────────────┐   │
│  │ 방이동            │   │
│  └──────────────────┘   │
│                         │
│  📝 비고                 │
│  ┌──────────────────┐   │
│  │                  │   │
│  └──────────────────┘   │
│                         │
│  ┌───────────────────┐  │
│  │    ✅ 저장하기     │  │
│  └───────────────────┘  │
└─────────────────────────┘
```
- 바텀시트 **60fps 스프링 애니메이션**으로 올라옴
- 채널 = 드롭다운 (4개 고정 옵션)
- 업체명 = 필수 (빈칸이면 저장 불가, 버튼 비활성)
- 시간 = 15분 단위 휠 또는 time input. **방문/미팅 시간** 기입
- 저장 → 바텀시트 닫힘 + 월뷰에 **즉시 점 추가** (낙관적 UI)

### 저장 시 데이터 흐름
```
[앱] 저장 클릭
  → service.bookMeeting(user, {
      date: "2026-04-19",
      time: "13:00",
      channel: "현수막",
      vendor: "○○치킨",
      region: "방이동",
    })
    ① repo.appendMeeting(spreadsheetId, row)
       → 앱_미팅예약 탭에 행 추가: [2026-04-19, 13:00, 현수막, ○○치킨, 방이동, (수식), ""]
       → F열 수식 자동계산: "4/19 ○○치킨 방이동 13시"
       → 영업관리 탭 수식 자동갱신:
           I열: TEXTJOIN으로 미팅예약기록 쌓임
           J열: 오늘미팅일정에 "○○치킨 방이동" 추가
           K열: 오늘미팅수 1 → 2 증가
    ② repo.calendar.insertEvent(accessToken, event)
       → 수강생 본인 Google Calendar에 이벤트 생성
       → summary: "○○치킨"
       → location: "방이동"
       → start: 2026-04-19T13:00 (Asia/Seoul)
       → end: 2026-04-19T14:00 (+1h)
       → reminder: 30분 전 팝업
    ③ repo.updateEventId(spreadsheetId, rowIndex, eventId)
       → G열에 캘린더 이벤트 ID 저장
  ← { ok: true, eventId: "abc123" }

[앱] 월뷰 4/19에 • 추가 + 토스트 "미팅 등록 완료 📅"

실패 시:
  ① 시트 쓰기 실패 → 롤백 + 에러 토스트 "저장 실패. 다시 시도해주세요"
  ② 시트 OK + 캘린더 실패 → 시트는 유지 (SSOT!) + 경고 토스트 "캘린더 동기화 실패. 일정은 기록됨"
```

### 영업관리 탭에서 보이는 최종 결과
```
영업관리 탭 — 10행 (날짜 = 4/19)
┌────┬─────────┬──────────────────────┬───────────────────┬────────┐
│ C  │ ...     │ I: 미팅예약기록       │ J: 오늘미팅일정    │ K: 수  │
├────┼─────────┼──────────────────────┼───────────────────┼────────┤
│4/19│         │ 4/19 ○○치킨 방이동 13시│ ○○치킨 방이동     │   2    │
│    │         │ 4/19 △△부동산 잠실 15시│ △△부동산 잠실     │        │
└────┴─────────┴──────────────────────┴───────────────────┴────────┘
  (I, J 셀 내에서 CHAR(10) 줄바꿈으로 쌓임. Wrap 서식 적용.)
```

## 인터랙션 상세

### 숫자 입력 (생산·컨택·미팅·계약)
```
   ┌─────────────────────────┐
   │  컨택   [ - ]  5  [ + ] │
   │                         │
   │  • [-] 탭 → 4           │
   │  • [+] 탭 → 6           │
   │  • 숫자 "5" 탭 → 키패드  │
   │    열림 → 직접 입력      │
   │  • 길게 누르면 연속 증감  │
   └─────────────────────────┘
```

### 낙관적 UI — 미팅 저장
```
시간축 →

[저장 클릭]                      [서버 응답]
    │                               │
    ├─ 즉시: 월뷰에 • 추가           │
    ├─ 즉시: 바텀시트 닫힘           │
    ├─ 즉시: 토스트 "저장 중..."      │
    │                               │
    │          성공 시 ──────────── ✅ 토스트 "미팅 등록 완료 📅"
    │          실패 시 ──────────── ❌ • 제거 + 토스트 "저장 실패"
```

### 에러 시나리오
| 상황 | 앱 반응 |
|---|---|
| 시트 쓰기 실패 | • 제거 + "저장 실패. 다시 시도해주세요" |
| 시트 OK + 캘린더 실패 | 시트 유지 + "캘린더 동기화 실패. 일정은 기록됨 ⚠️" |
| 네트워크 끊김 | "인터넷 연결을 확인해주세요" |
| 토큰 만료 | 자동 리프레시 → 실패 시 재로그인 유도 |
| 중복 날짜+시간+업체 | (Phase 1은 허용, Phase 2에서 경고) |
