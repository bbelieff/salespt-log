# 백엔드 데이터 모델 — 경영일지 시트 1:1 매핑

> **이 문서가 권위 SSOT**: 영업관리 컬럼 매핑(B=요일, C=날짜, D=채널, E~H=4지표)에서 다른 문서와 충돌 시 **이 문서가 정답**. sheet-structure.md는 이 문서에 맞춰 작성된다.

## 실제 Google Sheets 탭 구조

> ⚠️ **실제 시트 탭 이름 그대로 사용** (공백·번호·괄호 포함). API 호출 시에도 이 이름 그대로 써야 한다.

| 실제 탭 이름 | MVP 사용? | 역할 |
|---|---|---|
| `대시보드(자동작성)` | ✅ 읽기 전용 | 모든 지표의 최종 집계, 웹은 읽기만 |
| `01 영업관리` | ✅ 메인 | 8주 × 28행 블록 구조. 4지표 직접입력 + 미팅/수납 수식 집계 + 실적관리(수납) 통합 |
| `04 업체관리(앱자동작성용)` | ✅ 메인 | 1행=1미팅, 19컬럼. 웹 미팅 쓰기의 모든 대상 |
| `02 계약관리` | 🚫 MVP 미사용 | 계약 사후 서류 체크리스트. **후속**에서 영업관리와 연동 예정 |
| `03 DB관리` | 🚫 MVP 미사용 | 채널별 raw 매입/생산 log. **후속**에서 채널별 입력 UI + 시트 연동 예정 |

> 💡 앱의 5탭(컨택관리/일정·계약/캘린더/수납/DB관리) 중 일부는 시트의 02·03 탭과 짝을 이루지만, **MVP에서는 시트 연동 없이 UI만 존재**. 후속 로드맵에서 연동.

## 시트 컬럼 매핑 (`01 영업관리` 탭) ⭐

> ⚠️ **A 컬럼 없음**: `01 영업관리` 탭은 **B열부터 시작**.

매주 블록 구조: 하루 = 4행(채널별), 컬럼은 다음과 같이 그룹화됨:

| 컬럼 | 헤더 | 그룹 | 비고 |
|---|---|---|---|
| B | 요일 | (날짜) | 자동 |
| C | 날짜 | (날짜) | 자동 |
| D | 채널 | (날짜) | 매입DB / 직접생산 / 현수막 / 콜·지·기·소 |
| E | 생산건 수 | **컨택관리** | 채널별 (직접 입력) |
| F | 유입건 수 | **컨택관리** | 채널별 (직접 입력) |
| G | 컨택진행 수 | **컨택관리** | 채널별 (직접 입력) |
| H | 컨택성공 수 | **컨택관리** | 채널별 (직접 입력, = 미팅예약 건수) |
| I | 미팅예약 기록 | **컨택관리** | 수식 자동, 예약일 기준, 콤마 구분 |
| J | 오늘 미팅 일정 | **일정·계약관리** | 수식 자동, 미팅날짜 기준, 콤마 구분 |
| K | 오늘 미팅 수 | **일정·계약관리** | 수식 자동, COUNTIFS |
| L | 미팅 완료 수 | **일정·계약관리** | 수식 자동, 상태 IN ["계약","완료"] |
| M | 미팅사유 | **일정·계약관리** | ⭐ 수식 자동 (`04 업체관리(앱자동작성용)!M`에서 TEXTJOIN) |
| N | 계약건 수 | **일정·계약관리** | 수식 자동, 상태="계약" |
| O | 수임비 금액 | **일정·계약관리** | 수식 자동, 상태="계약" SUMIFS |
| P | 계약비고 | **일정·계약관리** | 수식 자동, "업체명, 수임비, 계약조건" 누적 |
| Q | 승인건 수 | **실적관리(수납)** | ⭐ 일별 직접 입력 (별도 수납관리 탭 없음) |
| R | 수납건 수 | **실적관리(수납)** | ⭐ 일별 직접 입력 |
| S | 수납금액 | **실적관리(수납)** | ⭐ 일별 직접 입력 (만원 단위) |
| T | 비고(기관, 접수내용) | **실적관리(수납)** | ⭐ 일별 직접 입력 |

> ⭐ **수납 데이터(Q~T)는 영업관리에 통합** (2026-04-27 결정). 별도 "수납관리" 탭은 존재하지 않으며, 1입금=1행 모델은 폐기. 일별 합계만 영업관리 매주 블록에 직접 기록.

## 상단 요약 (1-6행) — 자동 계산

- **E~F열 1-6행**: 생산/유입/컨택/컨택성공/미팅/계약 총합 (`=SUM(...)`)
- **G~H열 1-6행 [영업효율관리]**: 컨택>일정/일정>미팅/미팅>계약/컨택>계약 성공률
- **H~L열 2-6행 [DB효율관리]**: 채널별 생산수/유입수/효율%
- **M~N열 2-6행 [성과관리]**: 수임비/승인수수료/매출 총합
- **N1**: 수강시작일, **N2**: 수료일

## 미팅 상태 enum (5가지) ⭐

미팅의 라이프사이클은 **5가지 상태**로 표현된다. UI 색상은 [tokens.md](./tokens.md), 시트 컬럼 매핑은 [sheet-structure.md](./sheet-structure.md) 참조.

| 상태 | 설명 | 좌측바 색 | 아이콘 | `04 업체관리!M` 작성 |
|---|---|---|---|---|
| `예약` | 액션 미선택 (처리 대기) | amber | 🟡 | — |
| `계약` | 미팅 후 계약 체결 (가장 좋은 결과) | green | 💵 | 안 적음 |
| `완료` | 미팅했으나 계약 X (사유 필수) | orange | 🟠 | `업체명, 이유` |
| `변경` | 일정 변경됨 (이 카드는 무효, 새 카드 생성) | purple | 📅 | `업체명, 새날짜로 변경` |
| `취소` | 취소·노쇼 (사유 필수) | red | 🔴 | `업체명, 이유` |

**중요 규칙**:
- `계약`/`완료`/`변경`/`취소`로 종결된 카드는 **`예약`으로 되돌리기 가능** (revert)
- `변경` 처리 시 기존 카드는 `상태='변경'`이 되고, **새 미팅 카드가 새 날짜에 자동 생성**된다 (업체·장소·예약비고 복사)
- `04 업체관리!K (계약여부)`는 `J열 상태='계약'`과 의미 중복이지만, 기존 수식 호환을 위해 유지 (계약 시 `K=TRUE` 동시 기록)

## 백엔드 데이터 모델 (TypeScript)

> ⚠️ **컬럼 표기 규칙**: 같은 알파벳이 여러 탭에 있어서 모호함을 방지하기 위해 항상 **`탭이름!컬럼` 형식**으로 명시. 예: `04 업체관리!M` (원본), `01 영업관리!M` (자동 집계).

```typescript
// lib/types/daily.ts
type ChannelKey = '매입DB' | '직접생산' | '현수막' | '콜·지·기·소';

// 미팅 상태 5가지
type MeetingStatus = '예약' | '계약' | '완료' | '변경' | '취소';

// 한 채널 한 행 (01 영업관리!E~H)
type ChannelRow = {
  channel: ChannelKey;
  생산: number;        // 01 영업관리!E
  유입: number;        // 01 영업관리!F
  컨택진행: number;    // 01 영업관리!G
  컨택성공: number;    // 01 영업관리!H, = 그 채널의 미팅예약 슬롯 수
};

// 일별 실적(수납) — 별도 탭 아님, 영업관리에 통합 (2026-04-27 결정)
type DailyRevenue = {
  승인건수: number;     // 01 영업관리!Q
  수납건수: number;     // 01 영업관리!R
  수납금액: number;     // 01 영업관리!S, 만원 단위
  비고: string;        // 01 영업관리!T (기관, 접수내용)
};

// 미팅 1건 (04 업체관리(앱자동작성용) 탭의 한 행)
type Meeting = {
  id: string;          // UUID                            (04 업체관리!A)
  예약일: string;       // YYYY-MM-DD                       (04 업체관리!B)
  예약시각: string;     // HH:MM                            (04 업체관리!C)
  미팅날짜: string;     // YYYY-MM-DD                       (04 업체관리!D)
  미팅시간: string;     // HH:MM, 15분 단위 강제             (04 업체관리!E)
  channel: ChannelKey; //                                  (04 업체관리!F)
  업체명: string;       //                                  (04 업체관리!G)
  장소: string;        //                                  (04 업체관리!H)
  예약비고: string;     // 미팅 전 준비정보                  (04 업체관리!I)

  // 미팅 후 업데이트
  상태: MeetingStatus;  // 5가지                            (04 업체관리!J)
  계약여부: boolean;    // J="계약"과 동기화                (04 업체관리!K)
  수임비: number;       // 만원, 계약 시만                  (04 업체관리!L)
  미팅사유: string;     // ⭐ `업체명, 이유` 형식           (04 업체관리!M)
                        //   01 영업관리!M으로 자동 집계됨
                        //   [완료]/[변경]/[취소] 시만 작성. [계약]은 안 적음.
  계약조건: string;     // ⭐ 계약 시만                     (04 업체관리!P)

  // 시트 수식 생성 (읽기 전용)
  표시상세: string;     // "4/25, 13:00, ○○치킨, 방이동"   (04 업체관리!N)
  표시요약: string;     // "13:00, ○○치킨, 방이동"         (04 업체관리!O)
  계약합성라인: string; // "○○부동산, 300, 6개월 분할"     (04 업체관리!Q, 계약 행만)
};

// 한 날짜의 일별 데이터 (01 영업관리 한 묶음 = 채널별 4개 + 실적 1개)
type DailyEntry = {
  date: string;        // YYYY-MM-DD (PK)
  요일: string;
  channels: ChannelRow[];   // 4개 (영업관리!E~H, 채널별)
  revenue: DailyRevenue;    // 실적관리 (영업관리!Q~T, 일별 1개)
  // 영업관리!I~P는 시트 수식 자동 집계, DailyEntry에는 미포함
  // (별도 SummaryView에서 읽어옴)
};

// API에서 한 날짜 종합 응답
type DailyView = {
  daily: DailyEntry;
  meetings: Meeting[];     // 04 업체관리에서 그 날짜 미팅 가져옴
  summary: {               // 상단 요약 (자동 계산)
    생산총합: number;
    유입총합: number;
    컨택총합: number;
    컨택성공총합: number;
    미팅총합: number;
    계약총합: number;
    수임비총합: number;
    매출총합: number;
    채널효율: { [k in ChannelKey]: { 생산: number; 유입: number; 효율: number } };
    영업효율: {
      컨택일정: number;    // H/F
      일정미팅: number;    // L/H
      미팅계약: number;    // N/L
      컨택계약: number;    // N/H
    };
  };
};
```

## 컨택관리 화면의 셀(Cell) 구조 (참고)

⚠️ 이 구조는 **프론트엔드 in-memory 상태**용이며, 시트 저장 시 위 `Meeting` 타입으로 변환된다.

```typescript
// 컨택관리 화면: 4채널 × 7요일 × 8주 = 셀 단위
// key 패턴: "{주차}_{요일idx}_{채널id}"
type Cell = {
  prod: number;      // → 01 영업관리!E: 생산
  inflow: number;    // → 01 영업관리!F: 유입
  contact: number;   // → 01 영업관리!G: 컨택진행
  success: number;   // → 01 영업관리!H: 컨택성공
  slots: Slot[];     // 미팅예약 카드 (success 수와 자동 동기화)
};

// 검증 규칙: success ≤ contact (위반 시 자동 보정)
// 검증 규칙: slots.length === success (자동 sync)
// 규칙 없음: inflow ≥ contact (전날 유입분을 오늘 컨택 가능)

type Slot = {
  id: string;                // UUID
  saved: boolean;            // [등록] 클릭 여부 (UI only, 시트 미저장)
  meetingDate: string;       // → 04 업체관리!D 미팅날짜
  meetingTime: string;       // → 04 업체관리!E 미팅시간 (15분 단위)
  company: string;           // → 04 업체관리!G 업체명
  location: string;          // → 04 업체관리!H 장소
  reservationNote: string;   // → 04 업체관리!I 예약비고
  status: MeetingStatus;     // → 04 업체관리!J 상태 (기본 '예약')
  contracted: boolean;       // → 04 업체관리!K 계약여부
  fee: number;               // → 04 업체관리!L 수임비
  meetingReason: string;     // → 04 업체관리!M 미팅사유
  // 계약조건(04 업체관리!P)은 일정·계약 화면에서만 입력
};
```

## 탭별 데이터 출처 (MVP)

| 탭 | 직접 쓰기 대상 | 수식 집계 | 역할 |
|---|---|---|---|
| `대시보드(자동작성)` | 없음 | ← 01 영업관리 | ❌ 읽기 전용 |
| `01 영업관리` | E~H (4지표), Q~T (실적) | I~P (자동 집계) | ✅ 메인 입력 + 자동 집계 |
| `04 업체관리(앱자동작성용)` | A~M, P (1미팅=1행) | N, O, Q (표시문자열·합성) | ✅ 미팅 원본 |
| `02 계약관리` | — | — | 🚫 MVP 미사용 (후속) |
| `03 DB관리` | — | — | 🚫 MVP 미사용 (후속) |

## API 엔드포인트 (예정)

```
GET  /api/daily/:date                    → DailyView (01 영업관리 탭 읽기)
POST /api/daily/:date                    → 채널 4지표 + 실적 저장 (01 영업관리!E~H, Q~T)
POST /api/meeting                        → 미팅 1건 추가 (04 업체관리 탭 append)
PATCH /api/meeting/:id                   → 미팅 상태/계약/수임비/사유/계약조건 업데이트
                                           (04 업체관리!J, K, L, M, P)
POST /api/meeting/:id/action             → 4가지 액션 (contract/done/reschedule/cancel)
                                           reschedule는 새 행 자동 생성
GET /api/meetings?date=YYYY-MM-DD&dateType=reservation|meeting → 미팅 조회
GET  /api/summary                        → 상단 요약 (대시보드 사용)
GET  /api/schedule                       → 수강시작일/수료일 + 주차 정보
```

> ⚠️ **변경**: `/api/payment` 엔드포인트 **제거**. 수납은 `/api/daily/:date`의 `revenue` 필드로 통합.

```
        ↓
[04 업체관리(앱자동작성용)] ↔ lib/repo/sheets.ts ↔ lib/service/meeting.ts ↔ /api/meeting ↔ React Query
        ↓ (시트 수식 자동 집계)
[01 영업관리 I~P]                                  ← 채널 4지표(E~H) 직접 쓰기
                                                  ← 실적(Q~T) 직접 쓰기 ⭐
        ↓ (시트 수식)  
[대시보드(자동작성)] ← 읽기 전용
```

### 주요 변경사항
- **웹 쓰기**: `04 업체관리` (미팅) + `01 영업관리` E~H·Q~T (4지표·실적)
- **수식 집계**: 영업관리 I~P 전체 ← 04 업체관리
- **캐시 키**: `['meetings', date]`, `['daily', date]` 분리
- **무결성**: 1미팅=1행으로 개별 편집 가능
- **상태 5개**: 예약/계약/완료/변경/취소
- **수납**: 별도 탭/모델 없음. 영업관리 일별 통합
