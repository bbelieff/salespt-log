### 주요 인터랙션과 API 호출
- **날짜 네비 (◀▶)**: `GET /api/daily/:date` → 해당 날짜 데이터 로드
- **채널 탭 클릭**: 로컬 상태 변경 (API 호출 없음)
- **숫자 스테퍼 (+/-)**: 로컬 상태 즉시 업데이트
- **숫자 직접 클릭**: 키패드 표시, 입력 완료 시 로컬 상태 업데이트
- **미팅 추가**: 미팅 예약 바텀시트 열기 (컨택성공 > 0일 때만 활성)
- **저장하기**: `POST /api/daily/:date` → 성공 시 XP 토스트, 실패 시 에러 토스트

### 상태 유지 규칙
- ✅ **탭 간 이동**: 입력 중인 데이터는 localStorage에 임시 저장
- ✅ **날짜 변경**: 미저장 데이터가 있으면 "저장하지 않은 변경사항이 있습니다" 확인 모달

## 5. 캘린더 탭 - 월간 시각화 🆕

> ❌ **확정 시안 미정**. 본 섹션이 임시 SSOT.
>
> **역할 재정의 배경**: v1까지 일정·계약 탭이 월 캘린더를 포함했으나, "이번 주 처리"와 "월 전체 시각화"를 분리하는 게 모바일 UX에 맞다고 판단 → 캘린더를 별도 탭으로 분리.

### 핵심 의도
- 월 단위로 미팅 분포를 시각적으로 파악
- 특정 날짜 클릭 → 일정·계약 탭의 해당 주로 점프
- 평소엔 사용 빈도 낮음 (일정·계약 탭이 주요 진입점)

### 모바일 레이아웃 (Primary)

```
┌─────────────────────────────┐
│ ◀  2026년 4월  ▶            │ 헤더: 월 네비 (수강 기간 내만)
├─────────────────────────────┤
│ 일 월 화 수 목 금 토         │ 요일 헤더 (일·토 빨강)
├──┬──┬──┬──┬──┬──┬──────────┤
│  │  │  │  │  │  │  1   2  │ 6주 그리드
│ 3│ 4│ 5│ 6│ 7│ 8│  9  10  │
│..│..│16│..│..│ 19●●●  20  │ 미팅 있는 날 점 표시 (채널 색)
│..│..│23●│..│..│  26  27  │ 오늘은 파란 배경
│  │  │  │  │  │           │
└──┴──┴──┴──┴──┴──┴──────────┤
│ 4월 23일 (목) - 오늘        │ 선택 날짜 미팅 리스트 (요약)
│ 🟡 14:00 ○○부동산 잠실      │
│ 💵 16:00 △△식당 방이동      │
│ → 일정·계약 탭으로 이동     │
└─────────────────────────────┘
```

### 셀 시각화 규칙
- **점**: 미팅 1건 = 점 1개 (최대 3개까지, 그 이상은 `+N`)
- **점 색**: 채널 색 (`매입DB=blue / 직접생산=green / 현수막=amber / 콜·지·기·소=purple`)
- **편집 가능 기간 외** (수강 시작 전, 수료+15일 이후): 셀 회색 + 클릭 비활성

### 인터랙션
- **셀 클릭** → 하단에 그 날 미팅 요약 표시
- **"일정·계약 탭으로 이동" 버튼** → 일정·계약 탭의 그 주차로 점프 + 그 날 day-section으로 스크롤
- **월 네비 ◀▶** → 수강 기간 내에서만 이동 (외부 월은 비활성)

### 시트 매핑
- **읽기만**: 업체관리 탭의 `D(미팅날짜), F(채널)` 컬럼 조회
- **쓰기 없음**: 캘린더 탭에서는 미팅 추가/수정/삭제 불가 (일정·계약 탭으로 유도)

### 향후 시안 작업 시 고려
- 점 vs 도트 vs 미니 카드 — 어느 시각화가 정보 밀도에 적절한지
- 6주 그리드 vs 5주 동적 — 모바일 세로 공간 절약
- 셀 길게 누르기 → 빠른 작업 메뉴 (Future)

## 공통 UI 패턴 및 가이드라인

### 1. 반응형 우선순위
- **Primary**: 모바일 (375px~768px) - 입력 중심 UI
- **Secondary**: PC/태블릿 (768px+) - 대시보드 중심 UI
- 하단 네비는 모바일에서만 표시, PC는 사이드바

### 2. 입력 컴포넌트 표준화
- **숫자 스테퍼**: 모든 수치 입력에 +/- 버튼 + 직접 입력
- **날짜**: 기본 HTML5 date picker + 네비게이션 화살표
- **드롭다운**: 네이티브 select (iOS/Android 최적화)
- **텍스트**: placeholder + 라벨로 명확한 가이드

### 3. 피드백 및 상태 표시
- **로딩**: 스켈레톤 UI 또는 스피너
- **성공**: 초록색 토스트 + 체크 아이콘
- **에러**: 빨간색 토스트 + 구체적 메시지
- **경고**: 노란색 토스트 + 해결 방법 안내

### 4. 접근성 (Accessibility)
- 터치 타겟 최소 44px × 44px
- 대비비 4.5:1 이상 (WCAG AA)
- 스크린 리더 지원 (aria-label, role)
- 키보드 네비게이션 지원 (PC)

### 5. 성능 최적화
- 무한 스크롤 대신 페이지네이션 (주문 내역, 수납 내역)
- 이미지 lazy loading
- API 응답 캐싱 (일별 데이터 5분, 요약 데이터 10분)
- 낙관적 UI로 체감 성능 향상
