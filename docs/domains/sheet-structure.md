# 시트 구조 정의

---

## 핵심 데이터 흐름 한눈에

```
[웹 컨택관리 탭]                  [웹 일정·계약 탭]
        ↓                                ↓
        └──────→ 업체관리 탭 (1미팅=1행, 모든 직접 쓰기) ←──────┘
                          ↓ (시트 수식 자동 집계)
                  영업관리 탭 (I,J,K,L,M,N,O,P)
                          ↓ (시트 수식)
                       대시보드 탭 (읽기 전용)
```

**중요**: 웹은 업체관리/수납관리에만 쓴다. 영업관리는 모두 시트 수식으로 자동 집계된다.

---

## 전체 탭 구조

| 순서 | 탭명 | 역할 | 데이터 흐름 |
|---|---|---|---|
| 1 | 대시보드 | 읽기 전용 집계 | ← 영업관리 (수식) |
| 2 | 영업관리 | 생산 직접입력 + 미팅/수납 수식집계 | E~H ← 웹 / I~T ← 수식 |
| 3 | **업체관리** | 미팅 1행=1건 저장 | ← 웹 (append/update) |
| 4 | **수납관리** | 수납 기록 저장 | ← 웹 (append) |
| 5 | 회고노트 | 자유 텍스트 | ← 웹 |

---

## 1. 대시보드 탭 (기존 유지)

**역할**: 모든 지표의 최종 집계. 웹은 읽기만, 쓰기 금지.

기존 수식 구조 유지 - 세부 컬럼 정의는 기존 문서 참조.

## 2. 영업관리 탭

### 직접 입력 컬럼 (웹 → 시트)

| 컬럼 | 필드 | 타입 | 소스 |
|---|---|---|---|
| A | 날짜 | date | 자동 |
| B | 요일 | weekday | 수식 |
| C | 일차 | number | 수식 |
| D | 채널 | 4종 | 폼 선택 |
| E | 생산 | number | 폼 입력 |
| F | 컨택 | number | 폼 입력 |
| G | 신규명함 | number | 폼 입력 |
| H | 팔로업 | number | 폼 입력 |

### 수식 집계 컬럼 (시트 → 시트, 모두 자동 집계)

| 컬럼 | 필드 | 기준 | 소스 탭 | 수식 패턴 |
|---|---|---|---|---|
| **I** | 미팅예약기록 | 예약일(B) + 채널(F) | 업체관리!N | TEXTJOIN+FILTER |
| **J** | 오늘미팅일정 | 미팅날짜(D) + 채널(F) | 업체관리!O | TEXTJOIN+FILTER |
| **K** | 오늘미팅수 | 미팅날짜(D) + 채널(F) | 업체관리 | COUNTIFS |
| **L** | 미팅완료수 | 미팅날짜(D) + 채널(F) + 상태 IN ["계약","완료"] | 업체관리!J | COUNTIFS |
| **M** | 미팅사유 | 미팅날짜(D) + 채널(F) | 업체관리!M | TEXTJOIN+FILTER ⭐ |
| **N** | 계약건수 | 미팅날짜(D) + 채널(F) + 상태="계약" | 업체관리!J | COUNTIFS |
| **O** | 수임비합계 | 미팅날짜(D) + 채널(F) + 상태="계약" | 업체관리!L | SUMIFS |
| **P** | 계약비고 | 미팅날짜(D) + 채널(F) + 상태="계약" | 업체관리!Q (합성) | TEXTJOIN+FILTER |
| **Q** | 승인건수 | 수납날짜(B) | 수납관리!C | SUMIFS |
| **R** | 수납건수 | 수납날짜(B) | 수납관리!D | SUMIFS |
| **S** | 수납금액 | 수납날짜(B) | 수납관리!E | SUMIFS |
| **T** | 수납내용 | 수납날짜(B) | 수납관리!F | TEXTJOIN+FILTER |

### 핵심 수식 예시

#### I열: 미팅예약기록 (예약일 기준 누적)
```
=TEXTJOIN(CHAR(10),TRUE, 
  FILTER(업체관리!N:N, 
    (업체관리!B:B=$C2) * (업체관리!F:F=$D2)))
```
**예시 출력** (한 셀):
```
4/25, 13:00, ○○치킨, 방이동
4/26, 10:00, △△마트, 잠실
```

#### J열: 오늘미팅일정 (미팅날짜 기준 누적)
```
=TEXTJOIN(CHAR(10),TRUE, 
  FILTER(업체관리!O:O, 
    (업체관리!D:D=$C2) * (업체관리!F:F=$D2)))
```
**예시 출력**:
```
13:00, ○○치킨, 방이동
15:30, □□카페, 홍대
```

#### L열: 미팅완료수 (계약 + 완료(계약X))
```
=COUNTIFS(업체관리!D:D, $C2, 업체관리!F:F, $D2, 업체관리!J:J, "계약")
+
COUNTIFS(업체관리!D:D, $C2, 업체관리!F:F, $D2, 업체관리!J:J, "완료")
```
> 변경/취소는 제외 (실제 미팅을 진행한 수만 집계).

#### M열: 미팅사유 (자동 누적) ⭐
```
=TEXTJOIN(CHAR(10),TRUE, 
  FILTER(업체관리!M:M, 
    (업체관리!D:D=$C2) * (업체관리!F:F=$D2)))
```
**예시 출력**:
```
○○부동산, 가격 협상 결렬 (다음주 재검토)
□□카페, 노쇼 (연락 두절)
◇◇마트, 4/30로 일정변경
```
> 업체관리!M에 적힌 미팅별 사유들이 미팅날짜+채널 기준으로 자동 합쳐짐. 업체별 한 줄씩, 줄바꿈으로 누적.

#### N열: 계약건수 (상태="계약" 기준)
```
=COUNTIFS(업체관리!D:D, $C2, 업체관리!F:F, $D2, 업체관리!J:J, "계약")
```

#### O열: 수임비합계 (자동 합산)
```
=SUMIFS(업체관리!L:L, 업체관리!D:D, $C2, 업체관리!F:F, $D2, 업체관리!J:J, "계약")
```

#### P열: 계약비고 ("업체명, 수임비, 계약조건" 누적)
```
=TEXTJOIN(CHAR(10),TRUE, 
  FILTER(업체관리!Q:Q, 
    (업체관리!D:D=$C2) * (업체관리!F:F=$D2) * (업체관리!J:J="계약")))
```
> 업체관리!Q는 합성 헬퍼 컬럼: `=G2&", "&L2&", "&P2`

**예시 출력**:
```
○○부동산, 300, 6개월 분할 부가세 별도
□□카페, 500, 일시불 결제
```

## 3. 업체관리 탭

**역할**: 1미팅 = 1행. 웹의 미팅 관련 모든 쓰기 작업 대상.

| 컬럼 | 필드 | 타입 | 입력 소스 | 설명 |
|---|---|---|---|---|
| **A** | id | UUID | 웹 자동 | 고유 식별자 |
| **B** | 예약일 | date | 웹 자동 | 예약을 잡은 날 (생산 지표) |
| **C** | 예약시각 | time | 웹 자동 | 예약 기록 시점 |
| **D** | 미팅날짜 | date | 폼 입력 | 실제 미팅 진행 날 (컨택 지표) |
| **E** | 미팅시간 | time | 폼 입력 | 미팅 진행 시간 (15분 단위 강제) |
| **F** | 채널 | 4종 | 폼 선택 | 매입DB / 직접생산 / 현수막 / 콜·지·기·소 |
| **G** | 업체명 | text | 폼 입력 | 미팅 대상 업체 |
| **H** | 장소 | text | 폼 입력 | 미팅 장소 |
| **I** | 예약비고 | text | 폼 입력 | 미팅 전 준비정보 (지참서류, 부재 시간 등) |
| **J** | 상태 | enum(5) | 웹 업데이트 | **예약 / 계약 / 완료 / 변경 / 취소** |
| **K** | 계약여부 | boolean | 미팅 후 | TRUE/FALSE (J="계약"과 동기화) |
| **L** | 수임비 | number | 계약 시 | 만원 단위 |
| **M** | 미팅사유 | text | 미팅 후 | ⭐ `업체명, 이유` 형식. 영업관리!M으로 자동 집계됨 |
| **N** | 표시_상세 | formula | 시트 수식 | 콤마 구분, 날짜 포함 |
| **O** | 표시_요약 | formula | 시트 수식 | 콤마 구분, 날짜 제외 |
| **P** | 계약조건 | text | 계약 시 | 계약 특이사항 (예: "6개월 분할, 부가세 별도") |
| **Q** | 계약합성라인 | formula | 시트 수식 | "업체명, 수임비, 계약조건" 합성 → 영업관리!P 소스 |

### J열: 미팅 상태 5가지 ⭐

| 상태 | 의미 | 진입 액션 | 추가 입력 | 영업관리 영향 |
|---|---|---|---|---|
| `예약` | 액션 미선택 (기본값) | 미팅 카드 생성 시 | — | I, J, K |
| `계약` | 미팅 후 계약 체결 | [💵 계약] | 수임비(L), 계약조건(P) 필수 | L+1, N+1, O 합산, P 누적 |
| `완료` | 미팅했으나 계약 X | [🟠 완료] | 사유(M) 필수, 형식 `업체명, 이유` | L+1, M 누적 |
| `변경` | 일정 변경됨 (이 카드 무효) | [📅 변경] | 새 날짜·시간 → **새 행 자동 생성** + 사유(M) | M 누적 (원래 날짜) |
| `취소` | 취소·노쇼 | [🔴 취소] | 사유(M) 필수, 형식 `업체명, 이유` | M 누적 (L엔 미포함) |

**상태 전이 규칙**:
- `예약` → `계약` / `완료` / `변경` / `취소` (4가지 액션)
- 종결 상태 4개 → `예약`으로 되돌리기 가능 (revert, M 기록은 보존)
- `변경` 시 기존 카드는 그대로 두고 (상태='변경'), 새 미팅 카드를 새 날짜에 append (업체·장소·예약비고 복사)

### M열: 미팅사유 누적 포맷 ⭐

**형식**: `업체명, 이유` (한 줄)

새 사유가 생기면 그 미팅 행의 M에 한 줄로 적는다. 1미팅=1행이므로 한 행의 M열에는 보통 한 줄만 들어간다.

여러 미팅의 사유는 영업관리!M에서 시트 수식으로 자동 합쳐짐 (미팅날짜+채널 기준).

**액션별 M열 작성 규칙**:

| 액션 | M열 작성 형식 |
|---|---|
| [계약] | M열 작성 안 함 (사유 없음) |
| [완료] | `업체명, 계약 못한 이유` |
| [변경] | `업체명, 새날짜로 변경` (예: `◇◇마트, 4/30 10:00로 변경`) |
| [취소] | `업체명, 취소 이유` |

**예시 (개별 행의 M열 셀)**:
```
○○부동산, 가격 협상 결렬 (다음주 재검토)
```

### Q열: 계약합성라인 (시트 수식) ⭐

업체관리에 추가되는 헬퍼 컬럼. 영업관리!P가 이걸 TEXTJOIN한다.

```
=IF(J2="계약", G2&", "&L2&", "&P2, "")
```
**예시 출력**: `○○부동산, 300, 6개월 분할 부가세 별도`

> J가 "계약"이 아니면 빈 문자열 → TEXTJOIN에서 자동 제외됨.

### 표시문자열 수식 (N, O열) ⭐ 콤마 구분

#### N열: 표시_상세 (날짜 포함)
```
=TEXT(D2,"M/d")&", "&TEXT(E2,"HH:MM")&", "&G2&", "&H2
```
**예시**: `4/25, 13:00, ○○치킨, 방이동`

#### O열: 표시_요약 (날짜 제외)  
```
=TEXT(E2,"HH:MM")&", "&G2&", "&H2
```
**예시**: `13:00, ○○치킨, 방이동`

## 사용자 시나리오 → 시트 기록 매핑 ⭐

### 시나리오 A: 미팅 예약 (컨택관리 탭)

1. 컨택성공 +1 → 미팅카드 생성
2. 일정·시간·업체·장소 입력 + (선택)예약비고
3. [등록] 클릭 → **업체관리에 1행 append** (J="예약")

**시트 자동 반응**:
- 영업관리!I (예약일+채널 기준) → 한 줄 추가됨
- 영업관리!J (미팅날짜+채널 기준) → 한 줄 추가됨
- 영업관리!K (미팅날짜+채널 기준) → +1

### 시나리오 B: [계약] 액션

1. 일정·계약 탭에서 미팅 카드 → [💵 계약]
2. 수임비 + 계약조건 입력 → [계약 확정]
3. **업체관리 해당 행 update**: J="계약", K=TRUE, L=수임비, P=계약조건

**시트 자동 반응**:
- 영업관리!N → +1
- 영업관리!O → 수임비 합산
- 영업관리!P → "업체명, 수임비, 계약조건" 한 줄 추가
- 영업관리!L → +1 (계약+완료 둘 다 카운트)

### 시나리오 C: [완료] 액션 (계약 못 함)

1. 일정·계약 탭 → [🟠 완료]
2. 계약 못한 이유 입력 → [완료로 처리]
3. **업체관리 해당 행 update**: J="완료", M=`업체명, 이유`

**시트 자동 반응**:
- 영업관리!L → +1
- 영업관리!M → `업체명, 이유` 한 줄 추가

### 시나리오 D: [변경] 액션

1. 일정·계약 탭 → [📅 변경]
2. 새 날짜·시간 입력 → [변경 확정]
3. **업체관리 2단계 처리**:
   - 기존 행 update: J="변경", M=`업체명, 새날짜로 변경`
   - **새 행 append**: J="예약", 새 날짜·시간, 업체·장소·예약비고 복사

**시트 자동 반응**:
- 영업관리!M (원래 날짜+채널) → `업체명, 새날짜로 변경` 한 줄 추가
- 영업관리!I/J/K (새 날짜+채널) → 자동 추가됨 (새 행 append 효과)

### 시나리오 E: [취소] 액션

1. 일정·계약 탭 → [🔴 취소]
2. 취소 이유 입력 → [취소 처리]
3. **업체관리 해당 행 update**: J="취소", M=`업체명, 이유`

**시트 자동 반응**:
- 영업관리!M → `업체명, 이유` 한 줄 추가
- 영업관리!L → 미포함 (취소는 미팅완료수에 안 들어감)

## API 매핑

### 쓰기 대상 탭
- `POST /api/meeting` → 업체관리 탭 append (A~M, P)
- `PATCH /api/meeting/:id` → 업체관리 탭 특정 행 update (J, K, L, M, P)
- `POST /api/meeting/:id/action` → 4가지 액션 처리 (contract/done/reschedule/cancel)
  - `reschedule`: 기존 행 update + **새 행 append**
- `POST /api/payment` → 수납관리 탭 append
- `POST /api/daily` → 영업관리 탭 E~H 컬럼

### 읽기 전용 탭
- 영업관리 I~T (수식 집계)
- 대시보드 전체 (수식 집계)

### 중요 제약
- ❌ 웹이 영업관리 I~T에 직접 쓰기 시도 시 린터 차단
- ❌ 웹이 대시보드에 쓰기 시도 시 구조 테스트 실패
- ✅ 업체관리/수납관리만 웹 직접 쓰기 허용

## 검증 기준

### 수식 정확성
- [ ] N열 표시_상세에 날짜 포함 + 콤마 구분 확인
- [ ] O열 표시_요약에 날짜 제외 + 콤마 구분 확인  
- [ ] I열(예약일 기준) ≠ J열(미팅날짜 기준) 동작 확인
- [ ] 채널별 필터링 정상 작동
- [ ] 영업관리!N(계약건수)이 `J="계약"` 기준으로 카운트되는지 확인
- [ ] 영업관리!L(미팅완료수) = 계약 + 완료(계약X) 합 확인
- [ ] 영업관리!M에 업체관리!M의 사유들이 미팅날짜+채널별로 자동 합쳐지는지 확인
- [ ] 영업관리!P에 업체관리!Q(합성)이 계약 행만 필터링되어 누적되는지 확인

### 데이터 무결성
- [ ] 1미팅 = 1행 보장
- [ ] UUID 고유성 확인
- [ ] 상태 전이 검증 — 5가지 상태 모두 진입/이탈 확인:
  - [ ] 예약 → 계약/완료/변경/취소 (4가지 액션)
  - [ ] 계약/완료/변경/취소 → 예약 (revert)
  - [ ] 변경 시 새 행 자동 생성 (업체·장소·예약비고 복사)
- [ ] M열에 액션별 형식대로 적히는지 확인 (`업체명, 이유`)
- [ ] J='계약'일 때 K=TRUE, L>0, P 비어있지 않음 (계약 무결성)

### 성능
- [ ] FILTER 수식 응답 속도 < 2초
- [ ] 행 추가 시 수식 자동 확장
