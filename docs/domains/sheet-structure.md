# 시트 구조 정의

> **이 문서의 권위**: 컬럼 매핑은 [data-model.md](./data-model.md)가 SSOT. 이 문서는 거기에 맞춘 시트 작업 가이드.

## 핵심 데이터 흐름 한눈에

```
[웹 컨택관리 탭]                  [웹 일정·계약 탭]
        ↓                                ↓
        └──→ 04 업체관리(앱자동작성용) (1미팅=1행, 모든 미팅 쓰기) ←──┘
                          ↓ (시트 수식 자동 집계)
                  01 영업관리 (I~P 자동 집계)
                  ↑ (직접 쓰기)
                  E~H (4지표) + Q~T (실적/수납)
                          ↓ (시트 수식)
                  대시보드(자동작성) [읽기 전용]
```

**중요**:
- 웹은 `04 업체관리`(미팅) + `01 영업관리`(4지표·실적)에만 쓴다
- `01 영업관리!I~P`는 시트 수식으로 자동 집계 (절대 직접 쓰지 않음)
- **별도 수납관리 탭은 존재하지 않음** — 수납은 `01 영업관리!Q~T`에 일별 통합

## 2. `01 영업관리` 탭 ⭐

> ⚠️ **A 컬럼 없음**: B열부터 시작. data-model.md와 일치.

### 직접 입력 컬럼 (웹 → 시트)

| 컬럼 | 필드 | 타입 | 그룹 | 소스 |
|---|---|---|---|---|
| B | 요일 | weekday | (날짜) | 자동 |
| C | 날짜 | date | (날짜) | 자동 |
| D | 채널 | 4종 | (날짜) | 폼 선택 |
| **E** | **생산** | number | 컨택관리 | 폼 입력 |
| **F** | **유입** | number | 컨택관리 | 폼 입력 |
| **G** | **컨택진행** | number | 컨택관리 | 폼 입력 |
| **H** | **컨택성공** | number | 컨택관리 | 폼 입력 (= 미팅예약 건수) |
| **Q** | **승인건수** | number | 실적관리(수납) ⭐ | 폼 입력 |
| **R** | **수납건수** | number | 실적관리(수납) ⭐ | 폼 입력 |
| **S** | **수납금액** | number | 실적관리(수납) ⭐ | 폼 입력 (만원) |
| **T** | **비고** | text | 실적관리(수납) ⭐ | 폼 입력 (기관, 접수내용) |

> ⚠️ **검증 규칙**:
> - `컨택성공 ≤ 컨택진행` (H ≤ G) — 위반 시 자동 보정
> - `컨택성공 = 04 업체관리에 그 채널/그날 예약 잡힌 미팅 수`
> - `유입 ≥ 컨택진행` 규칙 **없음** (전날 유입분을 오늘 컨택 가능)

> ⭐ **수납 데이터(Q~T)**: 별도 수납관리 탭은 없으며, 일별 합계만 영업관리에 직접 입력. 1입금=1행 모델은 폐기 (2026-04-27 결정).

### 수식 집계 컬럼 (시트 → 시트, 모두 자동 집계)

| 컬럼 | 필드 | 기준 | 소스 탭 | 수식 패턴 |
|---|---|---|---|---|
| **I** | 미팅예약기록 | 예약일(B) + 채널(F) | `04 업체관리(앱자동작성용)!N` | TEXTJOIN+FILTER |
| **J** | 오늘미팅일정 | 미팅날짜(D) + 채널(F) | `04 업체관리(앱자동작성용)!O` | TEXTJOIN+FILTER |
| **K** | 오늘미팅수 | 미팅날짜(D) + 채널(F) | `04 업체관리(앱자동작성용)` | COUNTIFS |
| **L** | 미팅완료수 | 미팅날짜(D) + 채널(F) + 상태 IN ["계약","완료"] | `04 업체관리!J` | COUNTIFS |
| **M** | 미팅사유 | 미팅날짜(D) + 채널(F) | `04 업체관리!M` | TEXTJOIN+FILTER ⭐ |
| **N** | 계약건수 | 미팅날짜(D) + 채널(F) + 상태="계약" | `04 업체관리!J` | COUNTIFS |
| **O** | 수임비합계 | 미팅날짜(D) + 채널(F) + 상태="계약" | `04 업체관리!L` | SUMIFS |
| **P** | 계약비고 | 미팅날짜(D) + 채널(F) + 상태="계약" | `04 업체관리!Q` (합성) | TEXTJOIN+FILTER |

> ⚠️ **수식 작성 시 주의**: 시트 함수에서 탭 이름에 공백·괄호가 있으면 **작은따옴표 wrapping 필수**. 예: `'04 업체관리(앱자동작성용)'!N:N`

### 핵심 수식 예시

#### I열: 미팅예약기록 (예약일 기준 누적)
```
=TEXTJOIN(CHAR(10),TRUE, 
  FILTER('04 업체관리(앱자동작성용)'!N:N, 
    ('04 업체관리(앱자동작성용)'!B:B=$C2) * 
    ('04 업체관리(앱자동작성용)'!F:F=$D2)))
```
**예시 출력**:
```
4/25, 13:00, ○○치킨, 방이동
4/26, 10:00, △△마트, 잠실
```

#### J열: 오늘미팅일정 (미팅날짜 기준 누적)
```
=TEXTJOIN(CHAR(10),TRUE, 
  FILTER('04 업체관리(앱자동작성용)'!O:O, 
    ('04 업체관리(앱자동작성용)'!D:D=$C2) * 
    ('04 업체관리(앱자동작성용)'!F:F=$D2)))
```

#### L열: 미팅완료수 (계약 + 완료(계약X))
```
=COUNTIFS('04 업체관리(앱자동작성용)'!D:D, $C2, 
          '04 업체관리(앱자동작성용)'!F:F, $D2, 
          '04 업체관리(앱자동작성용)'!J:J, "계약")
+
COUNTIFS('04 업체관리(앱자동작성용)'!D:D, $C2, 
         '04 업체관리(앱자동작성용)'!F:F, $D2, 
         '04 업체관리(앱자동작성용)'!J:J, "완료")
```

#### M열: 미팅사유 (자동 누적) ⭐
```
=TEXTJOIN(CHAR(10),TRUE, 
  FILTER('04 업체관리(앱자동작성용)'!M:M, 
    ('04 업체관리(앱자동작성용)'!D:D=$C2) * 
    ('04 업체관리(앱자동작성용)'!F:F=$D2)))
```
**예시 출력**:
```
○○부동산, 가격 협상 결렬 (다음주 재검토)
□□카페, 노쇼 (연락 두절)
◇◇마트, 4/30로 일정변경
```

#### N열: 계약건수 (상태="계약" 기준)
```
=COUNTIFS('04 업체관리(앱자동작성용)'!D:D, $C2, 
          '04 업체관리(앱자동작성용)'!F:F, $D2, 
          '04 업체관리(앱자동작성용)'!J:J, "계약")
```

#### O열: 수임비합계
```
=SUMIFS('04 업체관리(앱자동작성용)'!L:L, 
        '04 업체관리(앱자동작성용)'!D:D, $C2, 
        '04 업체관리(앱자동작성용)'!F:F, $D2, 
        '04 업체관리(앱자동작성용)'!J:J, "계약")
```

#### P열: 계약비고
```
=TEXTJOIN(CHAR(10),TRUE, 
  FILTER('04 업체관리(앱자동작성용)'!Q:Q, 
    ('04 업체관리(앱자동작성용)'!D:D=$C2) * 
    ('04 업체관리(앱자동작성용)'!F:F=$D2) * 
    ('04 업체관리(앱자동작성용)'!J:J="계약")))
```
> `04 업체관리!Q`는 합성 헬퍼 컬럼 (다음 섹션 참고).

## 3. `04 업체관리(앱자동작성용)` 탭 ⭐

**역할**: 1미팅 = 1행. 웹의 미팅 관련 모든 쓰기 작업 대상. 19컬럼.

| 컬럼 | 필드 | 타입 | 입력 소스 | 설명 |
|---|---|---|---|---|
| **A** | id | UUID | 웹 자동 | 고유 식별자 |
| **B** | 예약일 | date | 웹 자동 | 예약을 잡은 날 (생산 지표) |
| **C** | 예약시각 | time | 웹 자동 | 예약 기록 시점 |
| **D** | 미팅날짜 | date | 폼 입력 | 실제 미팅 진행 날 (컨택 지표) |
| **E** | 미팅시간 | time | 폼 입력 | 미팅 진행 시간 (15분 단위 강제) |
| **F** | 채널 | 4종 | 폼 선택 | 매입DB / 직접생산 / 현수막 / 콜·지·기·소 |
| **G** | 업체명 | text | 폼 입력 | 미팅 대상 업체 |
| **H** | 장소 | text | 폼 입력 | 미팅 장소 |
| **I** | 예약비고 | text | 폼 입력 | 미팅 전 준비정보 (지참서류, 부재 시간 등) |
| **J** | 상태 | enum(5) | 웹 업데이트 | **예약 / 계약 / 완료 / 변경 / 취소** |
| **K** | 계약여부 | boolean | 미팅 후 | TRUE/FALSE (J="계약"과 동기화) |
| **L** | 수임비 | number | 계약 시 | 만원 단위 |
| **M** | 미팅사유 | text | 미팅 후 | ⭐ `업체명, 이유` 형식. `01 영업관리!M`으로 자동 집계됨 |
| **N** | 표시_상세 | formula | 시트 수식 | 콤마 구분, 날짜 포함 |
| **O** | 표시_요약 | formula | 시트 수식 | 콤마 구분, 날짜 제외 |
| **P** | 계약조건 | text | 계약 시 | 계약 특이사항 |
| **Q** | 계약합성라인 | formula | 시트 수식 | "업체명, 수임비, 계약조건" 합성 → `01 영업관리!P` 소스 |

### J열: 미팅 상태 5가지 ⭐

| 상태 | 의미 | 진입 액션 | 추가 입력 | 영업관리 영향 |
|---|---|---|---|---|
| `예약` | 액션 미선택 (기본값) | 미팅 카드 생성 시 | — | I, J, K |
| `계약` | 미팅 후 계약 체결 | [💵 계약] | 수임비(L), 계약조건(P) 필수 | L+1, N+1, O 합산, P 누적 |
| `완료` | 미팅했으나 계약 X | [🟠 완료] | 사유(M) 필수 | L+1, M 누적 |
| `변경` | 일정 변경됨 | [📅 변경] | 새 날짜·시간 → 새 행 자동 생성 + 사유(M) | M 누적 |
| `취소` | 취소·노쇼 | [🔴 취소] | 사유(M) 필수 | M 누적 (L엔 미포함) |

**상태 전이**:
- `예약` → `계약`/`완료`/`변경`/`취소` (4가지 액션)
- 종결 4상태 → `예약`으로 revert 가능 (M 기록 보존)
- `변경` 시: 기존 행 `J='변경'`으로 + 새 행 append (업체·장소·예약비고 복사)

### M열: 미팅사유 누적 포맷 ⭐

**형식**: `업체명, 이유` (한 줄)

| 액션 | M열 작성 형식 |
|---|---|
| [계약] | M열 작성 안 함 |
| [완료] | `업체명, 계약 못한 이유` |
| [변경] | `업체명, 새날짜로 변경` |
| [취소] | `업체명, 취소 이유` |

### Q열: 계약합성라인 (시트 수식) ⭐

```
=IF(J2="계약", G2&", "&L2&", "&P2, "")
```
**예시 출력**: `○○부동산, 300, 6개월 분할 부가세 별도`

### 표시문자열 수식 (N, O열)

#### N열: 표시_상세 (날짜 포함, 콤마 구분)
```
=TEXT(D2,"M/d")&", "&TEXT(E2,"HH:MM")&", "&G2&", "&H2
```
**예시**: `4/25, 13:00, ○○치킨, 방이동`

#### O열: 표시_요약 (날짜 제외, 콤마 구분)
```
=TEXT(E2,"HH:MM")&", "&G2&", "&H2
```
**예시**: `13:00, ○○치킨, 방이동`

## 5. `03 DB관리` 탭 — MVP 핵심 (비용·영업기회 raw log)

**역할**: 채널별 raw 데이터 입력. 시트의 합계/평균단가 수식이 자동 집계 → `01 영업관리!J3:L6`에서 채널별 생산총합·계약총합·계약당단가가 계산되어 대시보드까지 흐름.

**메인 목표** (사용자 정의):
- 채널별 **총비용**과 **평균단가** 모니터링
- 영업이익 = 수임비 합계 − (3개 비용 채널 합계)

**4 섹션 layout** (1행 = 헤더, 2행~ = 데이터):

| 섹션 | 컬럼 범위 | 컬럼 | 성격 |
|---|---|---|---|
| 매입DB | **B:G** | 구매일·업체명·개당단가·주문개수·주문금액(수식)·기타 | 비용 (단가×수량) |
| 직접생산(메타 등) | **I:N** | 날짜·소재·기간예산·생산개수·개당단가(수식=예산÷개수)·기타 | 비용 (예산 모델) |
| 현수막 | **P:V** | 날짜·업체명·도착일·개당단가·주문개수·주문금액(수식)·기타 | 비용 (단가×수량) |
| 콜·지·기·소 | **X:AD** | 구분(콜드콜/지인/기고객/소개)·접수일·대표자명·업체명·소개처·연락처·조건 | 영업기회 정보 (비용 X) |

H / O / W: spacer (비움). A: 비움.

**합계 행**: 각 섹션 데이터 마지막 다음에 시트가 자동으로 두는 합계/평균단가 행이 있음 — **앱은 합계 행을 절대 덮어쓰면 안 됨**. A 또는 첫 컬럼이 "합계" 텍스트인 행은 데이터 행으로 인식하지 않음.

**원칙**:
- 앱은 **raw 입력만** (각 셀에 사용자 입력값 넣기). 합계·평균단가 수식은 시트에 이미 박혀있으니 우리는 절대 안 만짐.
- 입력 row는 헤더 다음 첫 빈 row부터 채우거나, 합계 행 위에 행 insert.
- `01 영업관리!J3:L6` 자동 집계는 시트 수식이 처리하므로 앱은 신경 X.

**후속 (Phase 2 of `09-db-management` plan)**:
- 콜·지·기·소 섹션은 비용 추적이 아니라 영업기회 정보 → UI는 비용 3채널과 다른 형태

## 사용자 시나리오 → 시트 기록 매핑

### 시나리오 A: 미팅 예약 (컨택관리 탭)

1. 컨택성공 +1 → 미팅카드 생성
2. 일정·시간·업체·장소 입력 + (선택)예약비고
3. [등록] 클릭 → **`04 업체관리`에 1행 append** (J="예약")

**시트 자동 반응**:
- `01 영업관리!I` (예약일+채널) → 한 줄 추가
- `01 영업관리!J` (미팅날짜+채널) → 한 줄 추가
- `01 영업관리!K` (미팅날짜+채널) → +1

### 시나리오 B: [계약] 액션

1. 일정·계약 탭에서 미팅 카드 → [💵 계약]
2. 수임비 + 계약조건 입력 → [계약 확정]
3. **`04 업체관리` update**: J="계약", K=TRUE, L=수임비, P=계약조건

**시트 자동 반응**:
- `01 영업관리!N` → +1
- `01 영업관리!O` → 수임비 합산
- `01 영업관리!P` → "업체명, 수임비, 계약조건" 한 줄 추가
- `01 영업관리!L` → +1

### 시나리오 C: [완료] 액션 (계약 못 함)

1. 일정·계약 탭 → [🟠 완료]
2. 계약 못한 이유 입력 → [완료로 처리]
3. **`04 업체관리` update**: J="완료", M=`업체명, 이유`

**시트 자동 반응**:
- `01 영업관리!L` → +1
- `01 영업관리!M` → `업체명, 이유` 한 줄 추가

### 시나리오 D: [변경] 액션

1. 일정·계약 탭 → [📅 변경]
2. 새 날짜·시간 입력 → [변경 확정]
3. **`04 업체관리` 2단계**:
   - 기존 행 update: J="변경", M=`업체명, 새날짜로 변경`
   - **새 행 append**: J="예약", 새 날짜·시간, 업체·장소·예약비고 복사

### 시나리오 E: [취소] 액션

1. 일정·계약 탭 → [🔴 취소]
2. 취소 이유 입력 → [취소 처리]
3. **`04 업체관리` update**: J="취소", M=`업체명, 이유`

**시트 자동 반응**:
- `01 영업관리!M` → `업체명, 이유` 한 줄 추가
- `01 영업관리!L` → 미포함 (취소는 미팅완료수에 안 들어감)

### 시나리오 F: 일별 실적(수납) 입력 ⭐ 신규

1. 수납 탭(앱)에서 그날의 승인건수/수납건수/수납금액/비고 입력
2. **`01 영업관리!Q~T`에 일별 직접 쓰기** (별도 탭 없음)

> 💡 시안의 수납 탭이 어떻게 입력 UI를 구성할지는 시안 작업 시점에 결정. 데이터 모델은 일별 4개 값(Q/R/S/T) 단순.

## API 매핑

### 쓰기 대상
- `POST /api/meeting` → `04 업체관리(앱자동작성용)` append (A~M, P)
- `PATCH /api/meeting/:id` → `04 업체관리` 특정 행 update (J, K, L, M, P)
- `POST /api/meeting/:id/action` → 4가지 액션 (contract/done/reschedule/cancel)
  - `reschedule`: 기존 행 update + **새 행 append**
- `POST /api/daily/:date` → `01 영업관리` E~H (4지표) + Q~T (실적)

### 읽기 전용
- `01 영업관리!I~P` (수식 집계)
- `대시보드(자동작성)` (수식 집계)

### ❌ 제거된 엔드포인트 (v4)
- ~~`POST /api/payment`~~ → `/api/daily/:date`의 `revenue` 필드로 통합

### 탭 이름 표기 규칙
- 시트 API 호출 시 **실제 탭 이름 그대로** (공백·괄호 포함)
- 예: `'04 업체관리(앱자동작성용)'!A1:Q1000`
- 코드 상수는 `lib/config/SHEET_RANGES`에 정의 (이미 PR 1.5에서 정렬 완료)
