# 시트 구조 정의

## 전체 탭 구조

| 순서 | 탭명 | 역할 | 데이터 흐름 |
|---|---|---|---|
| 1 | 대시보드 | 읽기 전용 집계 | ← 영업관리 (수식) |
| 2 | 영업관리 | 생산 직접입력 + 미팅/수납 수식집계 | E~H,M ← 웹 / I~L,N~T ← 수식 |
| 3 | **업체관리** | 미팅 1행=1건 저장 | ← 웹 (append/update) |
| 4 | **수납관리** | 수납 기록 저장 | ← 웹 (append) |
| 5 | 회고노트 | 자유 텍스트 | ← 웹 |

## 1. 대시보드 탭 (기존 유지)

**역할**: 모든 지표의 최종 집계. 웹은 읽기만, 쓰기 금지.

기존 수식 구조 유지 - 세부 컬럼 정의는 기존 문서 참조.

## 2. 영업관리 탭 (역할 변경)

### 직접 입력 컬럼 (웹 → 시트)

> ⚠️ M열은 더 이상 직접 입력 아님 (수식으로 변경됨, 아래 표 참조).

| 컬럼 | 필드 | 타입 | 소스 |
|---|---|---|---|
| A | 날짜 | date | 자동 |
| B | 요일 | weekday | 수식 |
| C | 일차 | number | 수식 |
| D | 채널 | 4종 | 폼 선택 |
| E | 생산 | number | 폼 입력 |
| F | 컨택 | number | 폼 입력 |
| G | 신규명함 | number | 폼 입력 |
| H | 팔로업 | number | 폼 입력 |

### 수식 집계 컬럼 (시트 → 시트)

| 컬럼 | 필드 | 기준 | 소스 탭 | 수식 패턴 |
|---|---|---|---|---|
| **I** | 미팅예약기록 | 예약일(B) | 업체관리!N | `TEXTJOIN(CHAR(10),TRUE, FILTER(...))` |
| **J** | 오늘미팅일정 | 미팅날짜(D) | 업체관리!O | `TEXTJOIN(CHAR(10),TRUE, FILTER(...))` |
| **K** | 오늘미팅수 | 미팅날짜(D) | 업체관리 | `COUNTIFS(...)` |
| **L** | 미팅완료수 | 미팅날짜(D)+완료 | 업체관리 | `COUNTIFS(..., "완료")` |
| **M** ⭐ | 미팅결과·취소사유 | 미팅날짜(D) | 업체관리!P (미팅결과) | `TEXTJOIN(CHAR(10),TRUE, FILTER(...))` |
| **N** | 계약건수 | 미팅날짜(D)+계약=TRUE | 업체관리 | `COUNTIFS(..., TRUE)` |
| **O** | 수임비합계 | 미팅날짜(D) | 업체관리!L | `SUMIFS(...)` |
| **P** ⭐ | 계약내역 | 미팅날짜(D)+계약=TRUE | 업체관리 G·L·M | `TEXTJOIN`, 형식: "업체명, 수임비, 조건" |
| **Q** | 승인건수 | 수납날짜(B) | 수납관리!C | `SUMIFS(...)` |
| **R** | 수납건수 | 수납날짜(B) | 수납관리!D | `SUMIFS(...)` |
| **S** | 수납금액 | 수납날짜(B) | 수납관리!E | `SUMIFS(...)` |
| **T** | 수납내용 | 수납날짜(B) | 수납관리!F | `TEXTJOIN(CHAR(10),TRUE, FILTER(...))` |

### 핵심 수식 예시

#### I열: 미팅예약기록 (예약일 기준)
```
=TEXTJOIN(CHAR(10),TRUE, 
  FILTER(업체관리!N:N, 
    (업체관리!B:B=$C2) * (업체관리!F:F=$D2)))
```

#### J열: 오늘미팅일정 (미팅날짜 기준)
```
=TEXTJOIN(CHAR(10),TRUE, 
  FILTER(업체관리!O:O, 
    (업체관리!D:D=$C2) * (업체관리!F:F=$D2)))
```

#### M열: 미팅결과·취소사유 (미팅날짜 기준) ⭐
업체관리 탭의 `미팅결과(P)` 컬럼에서 누적된 [완료(계약X)/변경/취소] 메모를 줄바꿈으로 합쳐 옴.
```
=TEXTJOIN(CHAR(10),TRUE,
  FILTER(업체관리!P:P,
    (업체관리!D:D=$C2) * (업체관리!F:F=$D2) * (업체관리!P:P<>"")))
```
**예시 결과**:
```
[완료(계약X)] △△식당, 가격 협상 결렬
[취소] ◎◎스튜디오, 노쇼
```

#### P열: 계약내역 — "업체명, 수임비, 조건" 3개 필드 ⭐
계약(K=TRUE)된 미팅의 G(업체명) + L(수임비) + M(계약조건) 세 컬럼을 합성.
```
=TEXTJOIN(CHAR(10),TRUE,
  FILTER(
    업체관리!G:G&", "&업체관리!L:L&", "&업체관리!M:M,
    (업체관리!D:D=$C2) * (업체관리!F:F=$D2) * (업체관리!K:K=TRUE)))
```
**예시 결과**:
```
○○부동산, 300, 6개월 분할 부가세 별도
☆☆법인, 500, 일시불
```

## 3. 업체관리 탭 (신규)

**역할**: 1미팅 = 1행. 웹의 미팅 관련 모든 쓰기 작업 대상.

| 컬럼 | 필드 | 타입 | 입력 소스 | 설명 |
|---|---|---|---|---|
| **A** | id | UUID | 웹 자동 | 고유 식별자 |
| **B** | 예약일 | date | 웹 자동 | 예약을 잡은 날 (생산 지표) |
| **C** | 예약시각 | time | 웹 자동 | 예약 기록 시점 |
| **D** | 미팅날짜 | date | 폼 입력 | 실제 미팅 진행 날 (컨택 지표) |
| **E** | 미팅시간 | time | 폼 입력 | 미팅 진행 시간 |
| **F** | 채널 | 4종 | 폼 선택 | 매입DB / 직접생산 / 현수막 / 콜·지·기·소 |
| **G** | 업체명 | text | 폼 입력 | 미팅 대상 업체 |
| **H** | 장소 | text | 폼 입력 | 미팅 장소 (필수) |
| **I** | 예약비고 | text | 폼 입력 | 예약 시 메모 (선택) |
| **J** | 상태 | enum | 웹 업데이트 | 예약/완료/계약/변경/취소 (5종) |
| **K** | 계약여부 | boolean | 미팅 후 | TRUE/FALSE |
| **L** | 수임비 | number | 계약 시 | 만원 단위 |
| **M** | 계약조건 | text | 계약 시 | 영업관리!P열의 "조건" 필드로 들어감 |
| **N** | 표시_상세 | formula | 시트 수식 | 날짜 포함 표시 (영업관리 I열) |
| **O** | 표시_요약 | formula | 시트 수식 | 날짜 제외 표시 (영업관리 J열) |
| **P** ⭐ | 미팅결과 | text | 웹 자동 누적 | 완료(계약X)/변경/취소 시 prepend 누적. 영업관리 M열로 흐름 |

### 표시문자열 수식

#### N열: 표시_상세 (날짜 포함)
```
=TEXT(D2,"M/d")&" "&TEXT(E2,"HH:MM")&" "&G2&" "&H2
```
**예시**: `4/25 13:00 ○○치킨 방이동`

#### O열: 표시_요약 (날짜 제외)  
```
=TEXT(E2,"HH:MM")&" "&G2&" "&H2
```
**예시**: `13:00 ○○치킨 방이동`

#### P열: 미팅결과 (웹 자동 누적, 수식 X) ⭐
일정·계약 탭에서 [완료/변경/취소] 액션 시 웹이 prepend로 누적. **계약 액션은 P열에 안 적음** (계약은 K/L/M에 적힘).

**누적 형식**:
| 액션 | 형식 | 예시 |
|---|---|---|
| 완료(계약X) | `[완료(계약X)] 업체명, 사유` | `[완료(계약X)] △△식당, 가격 협상 결렬` |
| 변경 | `[변경] 업체명, 새날짜 새시간` | `[변경] ◇◇마트, 2026-04-24 10:00로 변경` |
| 취소 | `[취소] 업체명, 사유` | `[취소] ◎◎스튜디오, 노쇼` |

여러 건이 누적되면 줄바꿈으로 (최신이 위, prepend):
```
[취소] ◎◎스튜디오, 노쇼
[완료(계약X)] △△식당, 가격 협상 결렬
```

### 예시 데이터

| A | B | C | D | E | F | G | H | I | J | K | L | M | N | O | P |
|---|---|---|---|---|---|---|---|---|---|---|---|---|---|---|---|
| uuid-1 | 2026-04-20 | 09:15 | 2026-04-25 | 13:00 | 현수막 | ○○치킨 | 방이동 | 사장님 컨택완료 | 계약 | TRUE | 500 | 6개월 분할 부가세 별도 | 4/25 13:00 ○○치킨 방이동 | 13:00 ○○치킨 방이동 | (계약은 P 안 적음) |
| uuid-2 | 2026-04-20 | 14:30 | 2026-04-26 | 10:00 | 매입DB | △△마트 | 잠실 | 통화 후 일정조율 | 완료 | FALSE | 0 | - | 4/26 10:00 △△마트 잠실 | 10:00 △△마트 잠실 | `[완료(계약X)] △△마트, 가격 협상 결렬` |
| uuid-3 | 2026-04-22 | 11:30 | 2026-04-23 | 14:00 | 직접생산 | ◎◎스튜디오 | 한남 | - | 취소 | FALSE | 0 | - | 4/23 14:00 ◎◎스튜디오 한남 | 14:00 ◎◎스튜디오 한남 | `[취소] ◎◎스튜디오, 노쇼` |

## API 매핑

### 쓰기 대상 탭
- `POST /api/meeting` → 업체관리 탭 append
- `PATCH /api/meeting/:id` → 업체관리 탭 특정 행 update
- `POST /api/payment` → 수납관리 탭 append
- `POST /api/daily` → 영업관리 탭 E~H, M 컬럼

### 읽기 전용 탭
- 영업관리 I~L, N~T (수식 집계)
- 대시보드 전체 (수식 집계)

### 중요 제약
- ❌ 웹이 영업관리 I~L, N~T에 직접 쓰기 시도 시 린터 차단
- ❌ 웹이 대시보드에 쓰기 시도 시 구조 테스트 실패
- ✅ 업체관리/수납관리만 웹 직접 쓰기 허용

## 검증 기준

### 수식 정확성
- [ ] N열 표시문자열에 날짜 포함 확인
- [ ] O열 표시문자열에 날짜 제외 확인  
- [ ] I열(예약일 기준) ≠ J열(미팅날짜 기준) 동작 확인
- [ ] 채널별 필터링 정상 작동

### 데이터 무결성
- [ ] 1미팅 = 1행 보장
- [ ] UUID 고유성 확인
- [ ] 상태 전이 검증 (예약 → 완료/취소)

### 성능
- [ ] FILTER 수식 응답 속도 < 2초
- [ ] 행 추가 시 수식 자동 확장
