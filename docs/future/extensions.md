# 향후 확장 기능 (Future Extensions)

## 📦 MVP에서 제거된 기능들

### UI/UX 컴포넌트 (design 시스템에서 제거)

#### Modal (중앙 모달) - P2 단기
**제거 이유**: 8주 수강생에게 복잡한 모달 대화상자는 과잉 설계
**현재 대안**: Toast 알림으로 충분한 피드백 제공
**재도입 트리거**: 수료생 구독 전환 시 결제 모달, 복잡한 설정 필요 시
**기존 모델 영향**: 컴포넌트 추가만, 기존 구조 변경 없음

#### Alert (모달형 경고) - P2 단기  
**제거 이유**: Toast로 충분한 사용자 피드백, 모달형 경고는 과잉
**현재 대안**: `저장 완료!` 등 Toast 메시지
**재도입 트리거**: 중요한 데이터 손실 위험 상황 (결제 실패, 대량 삭제 등)
**기존 모델 영향**: 컴포넌트 추가만

#### Bottom Sheet - P3 중기
**제거 이유**: 미팅 추가는 간단한 인라인 폼으로 충분
**현재 대안**: Simple Form Card (같은 화면 내 폼)
**재도입 트리거**: 복잡한 폼이 필요해질 때 (고급 미팅 설정, 첨부파일 등)
**기존 모델 영향**: CSS 애니메이션 라이브러리 추가 필요

#### Icon Button (설정) - P2 단기
**제거 이유**: 8주 수강생에게 설정 기능이 불필요
**현재 대안**: 없음 (설정 자체를 제거)
**재도입 트리거**: 수료 후 개인화 설정 필요 시 (알림, 테마 등)
**기존 모델 영향**: 사용자 설정 테이블/API 추가 필요

#### Calendar Cell (월 캘린더) - P3 중기
**제거 이유**: 일정·계약 탭 자체가 MVP 범위 외
**현재 대안**: 주간 날짜 네비게이션
**재도입 트리거**: 월간 미팅 패턴 분석, 장기 일정 관리 필요 시
**기존 모델 영향**: 캘린더 렌더링 로직, 월별 데이터 집계 필요

### 반응형 범위 축소

#### 데스크톱 브레이크포인트 (md/lg) - P2 단기
**제거 이유**: 8주 수강생은 대부분 모바일 중심 사용
**현재 대안**: 375px 모바일 단일 버전
**재도입 트리거**: 데스크톱 사용자 비율 30% 초과 시
**기존 모델 영향**: CSS 반응형 규칙 추가, 컴포넌트별 데스크톱 레이아웃

#### text-3xl 타이포그래피 - P4 장기
**제거 이유**: "헬로 카드 메인 텍스트"가 MVP에 없음
**현재 대안**: text-2xl이 최대 크기
**재도입 트리거**: 대형 대시보드, 브랜딩 강화 필요 시
**기존 모델 영향**: 디자인 토큰만 추가

## 🚀 향후 기능별 우선순위

### P1 (즉시 후속, 1-2주) - 수료생 전환
- **수료 후 읽기전용 모드**: 데이터 조회만, 편집 불가
- **기본 구독 결제**: 단순 월/연 구독 모델
- **데이터 백업**: 수강 기간 데이터 보존

### P2 (단기 확장, 1-2개월) - 사용성 개선
- **Google Calendar 동기화**: 미팅 예약 연동
- **푸시 알림**: 브라우저 기본 알림
- **데스크톱 반응형**: md/lg 브레이크포인트
- **Modal/Alert 컴포넌트**: 결제, 중요한 경고용
- **사용자 설정**: 기본적인 개인화

### P3 (중기 확장, 3-6개월) - 플랫폼 확장
- **월간 캘린더**: 일정·계약 탭 본격 구현
- **Bottom Sheet**: 복잡한 폼, 첨부파일
- **월간/분기 리포트**: 기간 확장 분석
- **타 수강생 비교**: 랭킹, 벤치마킹 (옵션)
- **수료생 커뮤니티**: 기수별 그룹

### P4 (장기 확장, 6개월+) - SaaS화
- **일반 영업인 대상**: 수강생 아닌 일반 사용자
- **조직 기능**: 팀 관리, 멤버 초대
- **고급 분석**: AI 추천, 패턴 분석
- **다국어/접근성**: 글로벌 확장
- **고급 보안**: 2FA, 엔터프라이즈 보안

## 🎯 재도입 결정 가이드

### 언제 고려할 것인가
1. **사용자 요청**: 같은 기능을 3번 이상 요청받을 때
2. **사용량 임계점**: 특정 지표 도달 시 (예: 데스크톱 사용자 30%)
3. **비즈니스 모델**: 수익화, 확장을 위해 필수가 될 때
4. **경쟁 우위**: 차별화를 위해 반드시 필요할 때

### 재도입 시 고려사항
1. **기존 사용자 영향**: 현재 사용자 경험에 악영향 없는지
2. **유지보수 비용**: 추가 복잡도 vs 가치
3. **아키텍처 일관성**: 기존 설계 원칙과 충돌하지 않는지
4. **성능 영향**: 앱 로딩·렌더링에 부정적 영향 없는지
