# Playbook — 하네스 자체를 고치기 (Hashimoto 루프)

CLAUDE.md §0의 핵심 원칙: **에이전트가 실수를 하면, 같은 실수가 물리적으로 불가능해지도록 환경을 고친다.**

## 언제 실행하나
- 사용자가 같은 지적을 두 번 했을 때.
- 에이전트가 `--no-verify` 같은 우회를 시도했을 때.
- 테스트·훅·린터가 "잡았어야 할" 버그가 프로덕션에 새어 나갔을 때.

## 절차
1. **사건 요약**을 `docs/decisions/NNNN-harness-<slug>.md` 또는 incident 노트에 기록.
2. 이 실수를 막을 방법을 **아래 우선순위**로 고른다:
   1. 타입으로 막을 수 있는가?
   2. 린터 규칙으로 막을 수 있는가? (에러 메시지에 remediation 포함)
   3. 구조 테스트로 막을 수 있는가?
   4. pre-commit 훅으로 막을 수 있는가?
   5. `docs/playbooks/`에 절차로 박을 수 있는가?
   6. (최후) CLAUDE.md 규칙 한 줄 추가.
3. 고른 방법을 구현하고, **재현 테스트**(같은 실수를 의도적으로 시도하면 차단되는지)를 돌린다.
4. 원인을 Plan 문서가 아닌 **하네스 자체의 PR**로 커밋.

## 금지
- 에이전트 프롬프트에 "주의하세요" 추가로 끝내기. 그건 하네스 개선이 아님.
- 사건을 로그에서 삭제하기. 남겨야 복리로 학습함.
