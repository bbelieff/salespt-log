# ADR-0003: 업체관리·수납관리 탭 분리

> **📄 이 문서는 무엇인가요?**
> - **한 줄 요약**: 영업관리 탭의 미팅·수납 데이터를 별도 탭으로 분리하는 아키텍처 결정

## Context

### 문제 상황
기존 영업관리 탭의 I~P 컬럼(미팅/계약 기록)을 웹앱이 직접 작성할 때 **셀 내부 줄바꿈**으로 인한 데이터 무결성 문제가 발생했다:

1. **TEXTJOIN 수식 깨짐**: 미팅 여러 건을 한 셀에 `TEXTJOIN(CHAR(10), ...)` 으로 합칠 때, 웹에서 직접 문자열을 덮어쓰면 수식이 사라짐
2. **쿼리/정렬 한계**: 셀 안에 여러 미팅이 섞여 있어 개별 미팅 단위로 쿼리, 정렬, 통계 생성 불가
3. **데이터 일관성**: 같은 미팅 정보가 예약일 기준 I열과 미팅날짜 기준 J열 양쪽에 나타나야 하는데, 수동 중복 관리 시 불일치 발생

### 기술적 제약
- **Google Sheets API 한계**: `batchUpdate` 로 셀에 수식을 넣을 수 있지만, 기존 수식과 신규 데이터를 merge하는 로직이 복잡함
- **MVP 스코프**: 별도 DB 사용 불가, Google Sheets가 SSOT여야 함
- **정규화 필요**: 1미팅 = 1행 구조로 개별 CRUD 및 집계 쿼리 지원

---

## Decision

### 새로운 4탭 구조
기존 1개 탭(`01 영업관리`)을 4개 탭으로 분리:

1. **업체관리** (신규): 미팅 데이터의 SSOT
   - 1행 = 1미팅 (정규화)
   - 컬럼: A(ID) ~ M(계약비고)
   - 웹앱이 직접 `append` / `update`

2. **수납관리** (신규): 수납 데이터의 독립적 관리  
   - 1행 = 1일 수납 기록
   - 컬럼: A(ID) ~ F(기관비고)
   - 미팅과 분리된 별도 워크플로우

3. **영업관리** (기존, 역할 변경): 집계 전용 뷰
   - 기존 E~H(컨택), M(특이사항)만 직접 입력
   - I~L, N~T는 모두 시트 수식으로 자동 계산
   - 업체관리/수납관리 탭에서 데이터를 끌어와 표시

4. **대시보드** (기존): 읽기 전용 차트·지표

### 데이터 흐름
```
웹앱 입력
  ↓
업체관리 탭 (미팅) + 수납관리 탭 (수납)
  ↓ (시트 수식)
영업관리 탭 (집계 뷰)
  ↓ (시트 수식)  
대시보드 (차트·요약)
```

### 핵심 설계 원칙
- **I열 = 예약일 기준**, **J열 = 미팅날짜 기준**: 같은 미팅이 두 날짜의 다른 컬럼에 나타남 (시트 수식이 자동 처리)
- **업체 마스터 테이블 없음**: MVP에서는 같은 업체여도 매번 새 행 생성 (중복 허용)
- **웹 쓰기 vs 시트 계산 명확 분리**: 웹이 영업관리 I~L/N~T에 직접 쓰면 린터로 차단

## Consequences

### 긍정적 효과
1. **데이터 정규화**: 1미팅 = 1행으로 개별 CRUD, 정렬, 필터링 가능
2. **웹앱 단순화**: 복잡한 TEXTJOIN 로직 불필요, 단순한 append/update 패턴
3. **확장성**: 미팅별 상세 정보(참석자, 계약 조건 등) 컬럼 추가 용이  
4. **시트 수식 활용**: 기존 시트 사용자의 수식 지식을 그대로 활용, 집계는 시트가 담당
5. **분리된 관심사**: 미팅 관리와 수납 관리가 독립적 워크플로우로 분리

### 부정적 효과  
1. **수식 성능**: 업체관리 탭 행 수 증가 시 VLOOKUP/FILTER 수식 성능 저하 가능
2. **스키마 복잡도**: 4개 탭 간 데이터 일관성 유지를 위한 수식 관리 복잡
3. **시트 의존성**: 수식 오류 발생 시 웹앱에서 디버깅 어려움
4. **마이그레이션**: 기존 시트 데이터를 새 구조로 변환 필요

---

## Alternatives Considered

### A1: 웹에서 직접 TEXTJOIN 생성
- **아이디어**: 웹앱이 기존 I열 셀 내용을 읽어서 새 미팅을 TEXTJOIN 패턴으로 추가
- **기각 이유**: 
  - 기존 수식과 하드코딩된 문자열이 혼재하여 파싱 복잡
  - 동시성 이슈: 여러 미팅을 동시에 추가할 때 race condition
  - 롤백 어려움: 수식이 깨지면 복구 불가

### A2: 단일 탭에 미팅을 별도 영역(AA~ 컬럼)으로 분리  
- **아이디어**: 영업관리 탭의 오른쪽 영역을 미팅 전용으로 사용
- **기각 이유**:
  - 시트 가로 스크롤 문제: 사용자 경험 저하
  - 여전히 I~J열 중복 관리 이슈 해결 안됨

### A3: JSON 문자열로 셀에 저장
- **아이디어**: 미팅 배열을 JSON.stringify 하여 단일 셀에 저장
- **기각 이유**: 
  - 시트에서 수식으로 JSON 파싱 불가
  - 사용자가 시트에서 직접 읽을 수 없음

## Implementation Notes

### 마이그레이션 계획
1. 기존 시트의 I~P 컬럼 데이터를 파싱하여 업체관리 탭으로 이전
2. 영업관리 탭의 I~P 컬럼을 수식으로 교체  
3. 웹앱 API 엔드포인트 변경

### 모니터링 지표
- 업체관리 탭 행 수 vs 시트 수식 응답 시간
- 영업관리 탭 수식 오류 빈도
- 사용자 시트 직접 편집 vs 웹앱 사용 비율
