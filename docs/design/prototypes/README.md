# 화면별 확정 프로토타입

`docs/design/prototypes/` 는 Claude.ai Project에서 디자인이 **확정된 시안**을 보관하는 폴더입니다. Next.js 구현 시 "픽셀 퍼펙트 레퍼런스"로 사용합니다.

각 HTML 파일을 직접 열어 동작 확인:
```bash
npx serve docs/design/prototypes -l 5557
# → http://localhost:5557/contact-daily-input.html
# → http://localhost:5557/schedule-weekly.html
```

## 1. `contact-daily-input.html` — 컨택관리 탭 (v7)

### 의도
수강생이 **매일 한 번** 들어와서 **4채널의 4지표(생산/유입/컨택진행/컨택성공)를 입력**하고, 컨택성공이 발생하면 **그 자리에서 미팅 예약을 등록**까지 마치는 화면. 이 탭이 앱의 주된 진입점.

### 핵심 의사결정

| # | 결정 | 이유 |
|:-:|---|---|
| 1 | 주차 네비 = 금~목 (수강시작 요일 기준 동적) | scope.md의 "수강시작일 + 7일" 원칙. 하드코딩 금지 |
| 2 | 채널 4개를 **탭**으로 (한 번에 한 채널만 입력) | 모바일 가로 좁아서 4채널 동시 표시 불가. 탭에 합계 배지로 한눈에 |
| 3 | 입력 영역을 **2분할** (왼쪽 6 입력 / 오른쪽 4 합계) | "내가 지금 입력하는 채널 vs 오늘 전체"를 한 화면에서 비교 |
| 4 | 검증 규칙은 **컨택진행 ≥ 컨택성공만** | 유입은 어제 들어온 DB도 오늘 컨택 가능 → 유입≥컨택진행 강제하면 안 됨 |
| 5 | 컨택성공 cap: + 버튼 회색화 + 토스트 안내 | 직관적으로 "여기가 한계"를 보여줌 |
| 6 | 미팅 카드를 **신규(saved=false) / 등록완료(saved=true)** 로 분리 | 신규는 강제 펼침 + [등록] 버튼 / 등록완료는 한 줄 접힘 + 클릭 펼침 |
| 7 | 날짜 입력은 native date input을 0x0으로 숨기고 **커스텀 박스** + showPicker() | "YYYY-MM-DD (요일)" 표시가 가능해지고 picker UX는 native 그대로 |
| 8 | 시간은 **시 select(0~23) + 분 select(00·15·30·45)** | input type="time"의 step="900"은 모바일 picker가 무시함. select로 강제 |
| 8b | 신규 카드 등록 검증: 일정·시간·업체명·**장소 필수** | 시트에 빈 행 방지 (장소 필수는 2026-04-24 시트 사양 검증에서 추가) |
| 9 | 카드 [삭제] 시 채널의 컨택성공 -1 자동 | 슬롯 = 컨택성공이라는 1:1 매핑이 깨지지 않게 |
| 10 | 신규 카드 등록 검증: 날짜·시간·업체명 필수 | 시트에 빈 행 안 들어가게 |

### 시트 매핑

| 화면 | 시트 |
|---|---|
| 4채널 4지표 입력 | `영업관리!E~H` (채널별 4행) |
| 미팅 카드 [등록] | `업체관리!A~I` append (예약비고 = I열) |
| 미팅 카드 [수정] | `업체관리!A~I` update |
| 미팅 카드 [삭제] | `업체관리!` row delete + `영업관리!H` 컨택성공 -1 |

### 컨택관리 탭이 **하지 않는** 일
- 미팅 완료/계약/취소 처리 → **일정·계약 탭**에서
- 수임비 입력 → **일정·계약 탭**에서
- 미팅결과 메모 → **일정·계약 탭**에서

이 분리 원칙은 "예약 시점 / 미팅 후 시점"의 시간 차이 때문. 컨택관리 탭은 미팅 전 정보만 다룬다.

## 2. `schedule-weekly.html` — 일정·계약 탭 주간 뷰 (v2)

### 의도
이번 주(금~목)의 **모든 미팅을 한 화면에서 처리**. 미팅 후 결과(완료/계약/취소/변경)를 그 자리에서 기록. 월간 뷰는 별도 캘린더 탭이 담당.

### 5가지 카드 상태 (색상 체계 ⭐ 매우 중요)

| 상태 | 색 | 의미 | 다음 액션 |
|---|---|---|---|
| 🟡 **reserved** (예약) | 노랑 (`#fffbeb`) | 액션 미선택 — 처리 대기 | 4가지 액션 중 선택 |
| 💵 **contract** (계약) | 진초록 (`#dcfce7`) | 가장 좋은 결과 | 수임비·조건 표시 |
| 🟠 **done** (완료·계약X) | 주황 (`#fff7ed`) | 좋은 상태 아님 강조 | 사유 표시 |
| 📅 **rescheduled** (변경) | 보라 (`#faf5ff`) | 무효 + 정보 보존 | 새 카드가 변경된 날에 생성 |
| 🔴 **canceled** (취소) | 빨강 (`#fef2f2`) | 노쇼·무산 | 취소 사유 표시 |

> **노랑이 "예약 대기"인 이유**: 이전 v1은 흰색 배경이었으나, 노쇼·미처리 미팅을 시각적으로 알아채기 어려움. 노랑으로 "처리해야 할 일" 신호.

### 핵심 의사결정

| # | 결정 | 이유 |
|:-:|---|---|
| 1 | 7일을 **day-section** 박스로 묶음 (각 요일이 카드 컨테이너) | 어떤 미팅이 어느 날인지 한눈에 |
| 2 | 오늘 박스만 파랑 그라디언트 + 큰 그림자 | 시선 유도 |
| 3 | 미팅 카드 좌측 들여쓰기(18px) + 가지선(::before) | "이 날의 하위 항목" 시각 위계 |
| 4 | 4가지 액션 (계약/완료/변경/취소) **인라인 입력 폼** | 모달 X — 카드 안에서 바로 완결 |
| 5 | **변경**(reschedule) 시: 기존 카드 [변경] + 새 카드 자동 생성 | 일정 변경 이력 보존 + 미래 캘린더에 정확히 표시 |
| 6 | 미팅결과(시트 M열)는 **timestamp + 태그 prepend** 누적 | 시간순 역방향(최신 위) — 빠른 스캔 |
| 7 | 계약 시: 수임비(L열) + 계약조건(M열) 2개 분리 입력 | 영업관리 O열(SUMIFS) + P열(TEXTJOIN) 수식과 정확히 매핑 |
| 8 | "예약으로 되돌리기" 버튼 (모든 종결 상태에 있음) | 실수 복구 |
| 9 | 상단 요약 바: 전체/예약/완료/취소 카운트 + 수임비 합 | 주간 진척도 한눈에 |
| 10 | 요일 바 클릭 → 해당 day-section으로 부드러운 스크롤 | 빠른 점프 (scrollIntoView) |

### 시트 매핑 (2026-04-24 시트 사양 검증 후 갱신)

| 카드 액션 | 업체관리 탭 (1행=1미팅) | 영업관리 탭 (수식 자동) |
|---|---|---|
| 💵 계약 | `J=계약, K=TRUE, L=수임비, M=계약조건` | N열 +1 (COUNTIFS), O열 += 수임비 (SUMIFS), **P열 = "업체명, 수임비, 조건"** TEXTJOIN |
| 🟠 완료 | `J=완료`, **`P=[완료(계약X)] 업체명, 사유`** prepend | L열 +1 (COUNTIFS), **M열에 P열 누적분 TEXTJOIN** |
| 📅 변경 | 기존 `J=변경, P=[변경] 업체명, 새날짜 새시간` + 새 행 append (J=예약) | 변경된 날짜의 J열·K열에 자동 +1 |
| 🔴 취소 | `J=취소, P=[취소] 업체명, 사유` prepend | L열 카운트 **제외** (완료 아님), M열에 P열 누적분 |

> ⭐ **중요 변경**: 기존 시안은 `m.미팅결과` 필드에만 저장하고 영업관리 M열은 직접 입력 가정이었음.
> 사용자 사양 검증 결과 → **업체관리 탭에 P열(미팅결과) 신설 + 영업관리 M열은 P열에서 TEXTJOIN으로 자동**.

### 이 탭이 **하지 않는** 일
- 미팅 신규 등록 → **컨택관리 탭**에서 (컨택성공으로부터 슬롯 생성)
- 월간 시각화 → **캘린더 탭**에서 (별도)
- 컨택 4지표 입력 → **컨택관리 탭**

## 3. `calendar-monthly.html` — 캘린더 탭 월간 뷰 (v3) ⭐ UPDATED

### 의도
**"이번 주를 벗어나서 한 달 전체를 시각적으로 보고 싶을 때"** 사용. 평소 진입점은 일정·계약 탭이고, 캘린더는 점프 보조 도구. **읽기 전용 + 일정·계약 탭으로 점프**만 담당, 미팅 추가/수정/삭제 X.

### v3 주요 변경 (v1 대비)

| 항목 | v1 | v3 |
|---|---|---|
| 셀 비율 | 정사각형 (`1/1`) | **세로 직사각형 (`1/1.7`)** — 미팅 정보 표시용 |
| 셀 안 미팅 | 채널 점 3개 + `+N` | **시간 + 업체명 박스(pill) 3개 + `+N`** |
| 셀 표시 대상 | 모든 상태 | **변경/취소 제외** (예약/계약/완료만) |
| 채널 범례 | 점 + 라벨 | **배지 형태로 통일** (셀 안 박스와 시각 일관) |
| 컨택관리 아이콘 | 단순 박스 | **수화기 + 우상단 캘린더 + 일정 점 4개** |
| DB관리 아이콘 | 단순 실린더 | **카트 + 위에 DB박스** (DB 매입+생산 의미) |
| 하단 카드 | 모든 상태 | **모든 상태** (변경/취소 포함, 셀과 분리) |

### 핵심 의사결정

| # | 결정 | 이유 |
|:-:|---|---|
| 1 | 6주 × 7일 = 42칸 그리드, **셀 `aspect-ratio: 1/1.7`** | 미팅 시간+업체명을 셀 안에 직접 표시하려면 세로 공간 필요 |
| 2 | **셀 안 미팅 박스 = 채널 4색 (배경+글자) + 시간(굵게) + 업체명** | 점만으로는 "누구와 언제"를 못 알아봄. 상태는 색이 아니라 표시여부로 |
| 3 | 셀 안 박스는 최대 3개 + `+N` (예: `+2`) | 5개면 위 3개 표시 + 가운데 `+2` |
| 4 | **변경·취소 미팅은 셀에서 숨김**, 하단 카드에는 모두 표시 | 셀은 "유효한 미팅 한눈에", 하단은 "선택 날짜의 진실" |
| 5 | 박스는 미팅시간 오름차순 정렬 | 일과 흐름 그대로 |
| 6 | 오늘 = 파란 동그라미 배경 + 흰 숫자 + 그림자 | scope.md "오늘 강조" 원칙 |
| 7 | 선택 셀 = 파란 테두리 (`outline`) + 옅은 파란 배경 | 오늘 동그라미와 중첩 표시 가능 |
| 8 | 일·토 빨간 숫자 (요일 헤더 + 셀 모두) | 한국 캘린더 컨벤션 |
| 9 | 편집 가능 기간(수강시작~+69일) **외 셀 = 회색 + 클릭 비활성** | 사용자 룰 보호 |
| 10 | 다른 달 (그리드 채우기용) 셀 = 옅은 회색 + 클릭 비활성 | 깔끔한 그리드 + 잘못 클릭 방지 |
| 11 | 월 네비 ◀▶ 도 편집 기간 외로 못 가게 비활성 (opacity 0.25) | 기간 가드레일 일관성 |
| 12 | 채널 범례 = **배지** (셀 안 박스 색과 동일) | 색-이름 1:1 매핑 학습 즉시 |
| 13 | 선택 날짜 미니 카드는 **schedule-weekly와 같은 5상태 색** | 두 탭 시각 일관성 |
| 14 | 미니 카드 = 풀 카드 X (1줄로 압축) | 상세는 일정·계약 탭이 담당 |
| 15 | "→ 일정·계약 탭 N주차로 이동" 버튼에 주차 번호 표시 | 어디로 점프하는지 명확 |
| 16 | 안내 배너: 편집 외 셀이 보이는 월에만 노출 (조건부) | 노이즈 최소화 |
| 17 | **컨택관리 아이콘**: 수화기(전화) + 캘린더(일정) + 점 4개(DB 슬롯) | "전화로 컨택 → 일정 잡힘 → 4채널 관리" 의미 함축 |
| 18 | **DB관리 아이콘**: 카트 + 카트 위 DB박스 | "DB 매입(카트) + 생산(박스 채워가기)" 의미 함축 |

### 시트 매핑

| 캘린더 탭 동작 | 시트 작업 |
|---|---|
| 셀의 점 표시 | 업체관리 D(미팅날짜), F(채널) **읽기만** |
| 선택 미팅 미니 카드 | 업체관리 G/H/J/L/E **읽기만** |
| 셀/버튼 클릭 → 일정·계약 점프 | 시트 변경 없음 (탭 라우팅만) |

**쓰기 작업 0개**. 캘린더 탭은 **순수 읽기·점프 전용**.

### 이 탭이 **하지 않는** 일
- 미팅 추가/수정/삭제 → 일정·계약 탭으로 점프
- 컨택 입력 → 컨택관리 탭
- 월 단위 통계 → 대시보드 (별도)

### 보강 포인트 (시안에서 자체 강화된 부분)
- **주차 라벨**: 선택 날짜에 "1주차" 또는 "9주차 · 📌 마감 유예" 표시 (10주 마감 유예 가시화)
- **수임비 표시**: 계약 미팅의 수임비를 미니 카드에 inline 배지로 표시
- **취소 미팅** 시각: 카드에 line-through + 흐림(opacity 0.72)
- **편집 기간 외 클릭 시 토스트**: "편집 가능 기간이 아닙니다"

이 시안들의 의사결정 채팅이 일부 손실됐습니다. 위 표들이 **코드에서 역추출한 결정 기록**이며, 향후 비슷한 디자인 결정에 반복 참조 가능합니다.

코드 자체에 다음과 같은 형태로 결정 의도가 주석 처리되어 있어요:
- `/* 검증 규칙: 컨택진행 ≥ 컨택성공 (위반 시 컨택성공 자동 보정) — 유입 ≥ 컨택진행 규칙은 없음 — 전날 유입분을 오늘 컨택할 수 있기 때문 */`
- `/* native input type=date는 표시 형식을 바꿀 수 없으므로, 화면에는 우리가 만든 박스만 보여주고 native input은 0x0으로 숨김 */`
- `/* (입력 type=time의 step="900"은 모바일 picker가 무시하므로 select로 강제) */`

손실된 부분은 "거절된 대안들"과 "초기 시도 → 수정" 과정의 미세한 디테일 정도입니다.
